## 📚 МОДУЛЬ 1: ЧТО ТАКОЕ НЕЙРОСЕТЬ

### 1.1 АНАЛОГИЯ С МОЗГОМ

Нейросеть названа в честь биологических нейронов.

**Биологический нейрон:**
- получает сигналы от других нейронов (дендриты)
- если сигнал сильный — активируется
- передаёт сигнал дальше (аксон)

**Искусственный нейрон:**
- получает числа на вход
- умножает их на веса
- складывает
- пропускает через функцию активации
- выдаёт число на выход

### 1.2 ПРОСТЕЙШАЯ НЕЙРОСЕТЬ (ПЕРСЕПТРОН)

```
вход x1 → [умножить на вес w1] → сумматор → активация → выход
вход x2 → [умножить на вес w2] ↗
```

**Пример: предсказание погоды**
- x1 = температура (22°)
- x2 = влажность (65%)
- w1 = 0.7 (температура важна)
- w2 = 0.2 (влажность менее важна)
- сумма = 0.7×22 + 0.2×65 = 15.4 + 13 = 28.4
- активация: если сумма > 15 → «жарко»

**Что мы узнали:** нейросеть — это формула с весами.

## 📚 МОДУЛЬ 2: ИЗ ЧЕГО СОСТОИТ БОЛЬШАЯ НЕЙРОСЕТЬ

### 2.1 СЛОИ (LAYERS)

**Входной слой:** принимает данные (например, слова вопроса)

**Скрытые слои:** обрабатывают данные (их может быть 32, 80, 100)

**Выходной слой:** выдаёт ответ (например, слова ответа)

```
[вопрос] → [слой 1] → [слой 2] → [слой 3] → ... → [ответ]
```

### 2.2 НЕЙРОНЫ В СЛОЕ

В слое может быть 4096 нейронов. Каждый нейрон соединён с каждым нейроном следующего слоя.

**Количество соединений:**
- слой 1: 4096 нейронов
- слой 2: 4096 нейронов
- соединений: 4096 × 4096 = 16 777 216 (16 миллионов)

### 2.3 ВЕСА (WEIGHTS)

Каждое соединение имеет вес — число, которое определяет важность этого соединения.

В модели Llama 7B:

- 32 слоя
- 4096 нейронов в слое
- 7 миллиардов весов

**7 000 000 000 чисел** — это и есть «знание» модели.

### 2.4 ФУНКЦИИ АКТИВАЦИИ

Зачем: без них нейросеть была бы просто линейной формулой (складывать и умножать), что не позволяет решать сложные задачи.

**ReLU (Rectified Linear Unit):**
- если число положительное — оставить как есть
- если отрицательное — сделать нулём

**Softmax:**
- превращает числа в вероятности (сумма = 1)
- используется на выходном слое для выбора следующего слова

## 📚 МОДУЛЬ 3: КАК НЕЙРОСЕТЬ ПОНИМАЕТ ТЕКСТ

### 3.1 ТОКЕНИЗАЦИЯ

Нейросеть не понимает буквы. Она понимает числа.

**Токен** — кусочек текста (слово, часть слова, знак препинания)

Пример: «Что такое хесед?»
- «Что» → [1234]
- «такое» → [5678]
- «хесед» → [91011]
- «?» → [12]

**Словарь токенов:** 50 000 — 100 000 уникальных токенов.

### 3.2 ЭМБЕДДИНГИ (EMBEDDINGS)

Каждый токен превращается в вектор чисел (например, 4096 чисел).

«Царь» = [0.1, -0.3, 0.8, 0.0, ...] (4096 чисел)
«Король» = [0.09, -0.29, 0.81, 0.01, ...] — близкий вектор
«Стул» = [-0.5, 0.7, -0.1, 0.4, ...] — далёкий вектор

**Смысл:** близкие по смыслу слова имеют близкие векторы.

### 3.3 ВНИМАНИЕ (ATTENTION)

Самое важное изобретение (2017).

Позволяет модели смотреть на все слова одновременно и понимать, какие важнее.

**Пример:**
«Она сказала ему, что он умный»

Модель понимает, что «она» относится к действию «сказала», а «он» относится к «умный».

**Self-attention:** для каждого слова вычисляется, насколько оно связано с каждым другим словом.

## 📚 МОДУЛЬ 4: КАК НЕЙРОСЕТЬ ОБУЧАЕТСЯ

### 4.1 ПРЯМОЙ ПРОХОД (FORWARD PASS)

Вход (вопрос) → проходит через все слои → выход (ответ)

```
"Что такое хесед?" → [слой 1] → [слой 2] → ... → [слой 32] → "хесед — это..."
```

### 4.2 ФУНКЦИЯ ОШИБКИ (LOSS)

Сравниваем ответ модели с правильным ответом.

**Кросс-энтропия (Cross-Entropy Loss):**
- если ответ правильный → ошибка маленькая
- если ответ неправильный → ошибка большая

Числовой пример: правильный ответ «хесед», модель выдала «хесед» (0.9 уверена) → ошибка = 0.1

### 4.3 ОБРАТНЫЙ ПРОХОД (BACKPROPAGATION)

Ошибка передаётся обратно через все слои — от выхода ко входу.

На каждом слое вычисляется, как нужно изменить веса, чтобы ошибка уменьшилась.

**Цепное правило (Chain Rule):** математический способ распространять ошибку назад.

### 4.4 ГРАДИЕНТНЫЙ СПУСК (GRADIENT DESCENT)

Градиент — направление, в котором нужно изменить веса, чтобы ошибка уменьшилась.

Шаг обучения (learning rate) — насколько сильно менять веса.

```
новый_вес = старый_вес - learning_rate × градиент
```

### 4.5 ЭПОХИ (EPOCHS)

Одна эпоха = модель увидела все обучающие данные один раз.

Обычно нужно 3-10 эпох.

## 📚 МОДУЛЬ 5: АРХИТЕКТУРЫ НЕЙРОСЕТЕЙ

### 5.1 ПОЛНОСВЯЗНЫЕ СЕТИ (MLP)

Каждый нейрон соединён с каждым следующим.
Просто, но неэффективно для текста.

### 5.2 РЕКУРРЕНТНЫЕ СЕТИ (RNN)

Обрабатывают текст последовательно, слово за словом.
Запоминают предыдущие слова.
Проблема: забывают далёкие слова.

### 5.3 ТРАНСФОРМЕРЫ (TRANSFORMERS)

Современный стандарт (2017 — сейчас).
Основаны на механизме внимания.
Позволяют обрабатывать все слова одновременно.

**Llama, Qwen, Mistral, GPT — всё это трансформеры.**

### 5.4 ЧТО ТАКОЕ LLM (LARGE LANGUAGE MODEL)

Большая языковая модель — это трансформер с миллиардами весов, обученный на огромном количестве текста (интернет, книги).

Примеры: Llama 7B, 13B, 70B; GPT-3, GPT-4; Qwen; Mistral

## 📚 МОДУЛЬ 6: КАК НАСТРОИТЬ МОДЕЛЬ ПОД СЕБЯ

### 6.1 FINE-TUNE

Берём готовую обученную модель (Llama) и дообучаем на наших данных.

**Что меняется:** веса модели подстраиваются под нашу задачу.

**Что нужно:** 1000-2000 наших примеров, GPU

### 6.2 LORA (LOW-RANK ADAPTATION)

Метод эффективного fine-tune.

**Идея:** не менять все веса (это дорого), а добавить маленькие матрицы-адаптеры.

**Преимущества:**
- нужно в 100 раз меньше памяти GPU
- учится в 10 раз быстрее
- результат почти как у полного fine-tune

### 6.3 КВАНТИЗАЦИЯ (QUANTIZATION)

Уменьшаем точность весов для экономии памяти.

**Типы:**
- fp16 (16 бит на вес) — 14 GB для 7B модели
- q8_0 (8 бит) — 7 GB
- q4_k_m (4 бит) — 4-5 GB (минимальная потеря качества)
- q2_k (2 бита) — 2 GB (ощутимая потеря качества)

## 📚 МОДУЛЬ 9: ТЕРМИНОЛОГИЯ (ШПАРГАЛКА)

- **Вес (weight)** — число, определяющее важность связи между нейронами
- **Слой (layer)** — группа нейронов
- **Функция активации** — нелинейное преобразование
- **Токен** — единица текста (слово или часть слова)
- **Эмбеддинг** — векторное представление токена
- **Внимание (attention)** — механизм взвешивания важности слов
- **Forward pass** — проход от входа к выходу
- **Loss** — ошибка модели
- **Backpropagation** — распространение ошибки назад
- **Градиент** — направление изменения весов
- **Learning rate** — шаг изменения весов
- **Эпоха** — один проход по всем данным
- **Fine-tune** — дообучение готовой модели
- **LoRA** — метод эффективного дообучения
- **Квантизация** — сжатие весов (уменьшение точности)
- **GGUF** — формат сжатой модели для CPU
- **Инференс** — получение ответа от обученной модели

## 📚 МОДУЛЬ 10: ЧАСТЫЕ ВОПРОСЫ

**Вопрос:** Нужна ли математика?

**Ответ:** Для понимания — нет. Для реализации — да (линейная алгебра, производные, вероятность).

**Вопрос:** Можно ли обучить модель на обычном компьютере?

**Ответ:** Полный fine-tune нет (нужен GPU). LoRA с q4 моделью — можно на 32 GB RAM, но очень медленно.

**Вопрос:** Сколько стоит обучение?

**Ответ:** Аренда GPU на 1 день: $10-50. Бесплатно в Colab (с ограничениями).

**Вопрос:** Нужно ли учить математику?

**Ответ:** Если хочешь стать инженером — да. Если просто понимать принципы — достаточно этого курса.

**Вопрос:** Чем отличается GPT от Llama?

**Ответ:** Архитектура похожа. Разница в обучении (на каких данных), лицензии (Llama — открытая, GPT — закрытая), размере.

**Вопрос:** Можно ли обучить модель без кода?

**Ответ:** Можно через сервисы (Hugging Face AutoTrain, Google Vertex AI), но код даёт контроль.
