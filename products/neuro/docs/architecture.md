## ОСНОВНАЯ МОДЕЛЬ

Базовая архитектура: Llama 2 7B или Qwen 7B

Причина выбора:
- открытая лицензия
- хорошее качество на русском языке
- возможность запуска на обычном ПК (8-16 GB RAM)
- поддержка GGUF квантизации

## ТОНКАЯ НАСТРОЙКА (FINE-TUNE)

Метод: LoRA (Low-Rank Adaptation)

Параметры:
- rank = 16
- alpha = 32
- target modules = q_proj, v_proj

Обучаемые слои: ~0.5% от полной модели

## КВАНТИЗАЦИЯ

Тип: q4_k_m

Характеристики:
- размер: 4-5 GB
- потеря качества: минимальна (1-2%)
- скорость: в 2-3 раза быстрее fp16

## ПОТОК ДАННЫХ

1. Подготовка данных (prepare_data.py)
   - парсинг md-файлов
   - извлечение терминов, исследований, принципов
   - формирование пар (инструкция, ответ)

2. Обучение (train.py)
   - загрузка базовой модели
   - LoRA fine-tune на наших данных
   - сохранение адаптера

3. Квантизация (quantize.py)
   - слияние адаптера с базовой моделью
   - конвертация в GGUF
   - квантизация до q4_k_m

4. Инференс (server.py)
   - загрузка GGUF модели
   - запуск API сервера
   - обработка запросов

## ОЦЕНКА КАЧЕСТВА

Метрики:
- точность ответов (>90%)
- latency (<1 сек)
- размер модели (<5 GB)

## АЛЬТЕРНАТИВНЫЕ МОДЕЛИ

Если Llama 2 7B не подходит:
- Qwen 7B (лучше русский)
- Mistral 7B (быстрее)
- Gemma 7B (компактнее)

Решение: начать с Llama 2 7B, при проблемах — перейти на Qwen
