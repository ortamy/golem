**Метаданные файла**
- **Файл:** `instructions/self-learning-template.md`
- **Версия:** 1.0
- **Дата создания:** 2026-05-28
- **Статус:** Активный (будет пополняться)
- **Тема:** Шаблон для самостоятельного обучения нейросети работе с текстами ТаНаХа. Как нейросеть должна анализировать, проверять, разбирать и создавать исследования без участия человека.

---

## 🔥 ЦЕЛЬ

Научить нейросеть самостоятельно проходить полный цикл работы с любым текстом, содержащим ивритские понятия, библейские стихи или религиозные утверждения. Нейросеть должна уметь:

- Сканировать текст на религионимы
- Находить ивритские оригиналы
- Разбирать слова до корня
- Сверять с палео-ивритом
- Сравнивать переводы
- Находить искажения
- Писать исследование по шаблону
- Создавать логи проверки и полезности

---

## 📜 ПОРЯДОК РАБОТЫ

Нейросеть проходит 7 шагов. Каждый шаг обязателен.

---

### Шаг 1. ПОЛУЧЕНИЕ ТЕКСТА

**Вход:** любой текст на русском, английском или иврите.

**Действия:**
- Определи, содержит ли текст ивритские слова, цитаты из ТаНаХа или религиозные понятия
- Если да — переходи к Шагу 2
- Если нет — сообщи: «Текст не содержит материала для исследования»

---

### Шаг 2. ЛОГ ПРОВЕРКИ НА РЕЛИГИОНИМЫ

**Задача:** найти все слова, которые являются религионимами, грецизмами, церковнославянизмами или латинизмами.

**Как проверять:**
- Сверяй каждое слово с картой запрещённых слов (`forbidden-words.md`)
- Отмечай: ❌ нарушение, ⚠️ под вопросом, ✅ допустимо
- Учитывай контекст: если слово используется для разоблачения (в кавычках, с пометкой «синодальное») — ✅
- Если слово используется в авторской речи как подмена — ❌

**Формат вывода:**
```

ЛОГ ПРОВЕРКИ (עֵד) — [НАЗВАНИЕ ТЕКСТА]

Найденные фрагменты:
· ❌ «[слово]» — [тип нарушения]. Замена: «[правильное слово]».
· ⚠️ «[слово]» — [почему под вопросом]. Рекомендация.
· ✅ «[слово]» — [почему допустимо].

Итог: [краткий вывод]

```

---

### Шаг 3. ЛОГ ПОЛЕЗНОСТИ

**Задача:** оценить, что из текста можно извлечь для репозитория.

**Как оценивать:**
- Есть ли новые термины для папки `terminology/`?
- Есть ли новые темы для папки `researches/`?
- Есть ли новые принципы, методы или приёмы для `exposure-principles.md`?
- Есть ли новые слова для `forbidden-words.md`?
- Есть ли материал для `manifest.md`?

**Формат вывода:**
```

ЛОГ ПОЛЕЗНОСТИ (עֵד) — [НАЗВАНИЕ ТЕКСТА]

Что можно извлечь:
➕ Для terminology/: [список]
➕ Для researches/: [список]
➕ Для exposure-principles.md: [список]
➕ Для forbidden-words.md: [список]

Что уже есть: 🟡 [дубликаты]
Что не подходит: 🔴 [и почему]

Итог: [что создаём, что обновляем]

```

---

### Шаг 4. ИЗВЛЕЧЕНИЕ ИВРИТСКИХ СЛОВ

**Задача:** найти все ивритские слова в тексте и подготовить их к разбору.

**Как извлекать:**
- Выдели все ивритские слова с транслитерацией и переводом
- Если в тексте есть цитаты из ТаНаХа — сверь с масоретским текстом
- Если цитаты даны только в переводе — найди ивритский оригинал

**Формат вывода:**
```

ИВРИТСКИЕ СЛОВА ДЛЯ РАЗБОРА:
· [слово] ([транслитерация]) — «[перевод]». Корень: [Х.Х.Х]
· [слово] ([транслитерация]) — «[перевод]». Корень: [Х.Х.Х]

```

---

### Шаг 5. РАЗБОР ДО КОРНЯ

**Задача:** для каждого ивритского слова провести полный этимологический разбор.

**Как разбирать:**
- Определи трёхбуквенный корень
- Найди однокоренные слова в ТаНаХе
- Укажи родственные слова в других семитских языках (если есть)
- Дай буквальный перевод, избегая религиозной окраски

**Формат вывода:**
```

КОРНЕВЫЕ СВЯЗИ:
· [слово] — корень [Х.Х.Х]. «[значение корня]». Однокоренные: [список]

```

---

### Шаг 6. СВЕРКА С ПАЛЕО-ИВРИТОМ

**Задача:** разложить ключевые слова на буквы палео-иврита и собрать цепочку образов.

**Как сверять:**
- Для каждой буквы укажи палео-ивритский символ и его образ
- Собери цепочку образов в единую картину
- Сравни с корневым значением

**Формат вывода:**
```

СВЕРКА С ПАЛЕО-ИВРИТОМ:
[слово]:
· [Буква] ([символ]) — [образ].
· [Буква] ([символ]) — [образ].
Цепочка: «[образы через тире]». [Интерпретация].

```

---

### Шаг 7. НАПИСАНИЕ ИССЛЕДОВАНИЯ

**Задача:** создать md-файл по утверждённому шаблону (`research-template.md`).

**Обязательные блоки:**
1. Заголовок и метаданные
2. Введение
3. Оригинал (иврит + транслитерация + перевод)
4. Корневые связи
5. ТаНаХический контекст (3–5 стихов)
6. Сравнение переводов (Масоретский → Септуагинта → Вульгата → Синодальный)
7. Искажения (римско-христианский и талмудическо-раввинистический слои)
8. Возвращение

**Факультативные блоки:**
- Сверка с палео-ивритом
- Слова-двойники
- Исторический контекст
- Статистика употребления
- Связь с Йехошуа
- Обличение

**Правила написания:**
- Иврит + транслитерация + буквальный перевод для каждого стиха
- Без таблиц, только списки
- Без религионимов в авторской речи
- Эмодзи в заголовках разделов
- Имя файла: иврит латиницей, через дефис

## 📊 ПРИМЕР САМООБУЧЕНИЯ

### Входной текст
```

«В начале сотворил Бог небо и землю. Земля была безвидна и пуста, и Дух Божий носился над водою».

```

### Шаг 2. Лог проверки
```

❌ «Бог» — религионим. Замена: «Элохим».
❌ «Дух Божий» — религионим. Замена: «Дыхание Элохим».
✅ «небо», «земля», «вода» — не религионимы. Допустимо.

```

### Шаг 3. Лог полезности
```

➕ Для terminology/: «руах элохим» — дыхание Элохим.
➕ Для researches/: «берешит 1:1-2» — анализ первого стиха Торы.

```

### Шаг 4. Ивритские слова
```

· בְּרֵאשִׁית (бэрэшит) — «в начале». Корень: ר.א.ש
· אֱלֹהִים (Элохим) — «Всесильный». Корень: א.ל.ה
· רוּחַ (руах) — «дыхание, ветер». Корень: ר.ו.ח

```

### Шаг 5. Корневые связи
```

· בְּרֵאשִׁית — корень ר.א.ש. «Голова, начало». Однокоренные: רֹאשׁ (рош) — «голова», רִאשׁוֹן (ришон) — «первый».
· אֱלֹהִים — корень א.ל.ה. «Быть сильным». Однокоренные: אֵל (Эль) — «Сила», אֱלוֹהַּ (Элоhа) — «Божество».
· רוּחַ — корень ר.ו.ח. «Дуть, веять». Однокоренные: רֵיחַ (рэах) — «запах, дуновение», רֶוַח (рэвах) — «пространство, простор».

```

### Шаг 6. Сверка с палео-ивритом
```

רוּחַ (руах):
· Реш (𐤓) — голова. Начало, мысль.
· Вав (𐤅) — крюк. Соединение, связь.
· Хет (𐤇) — стена. Граница, отделение.
Цепочка: «Мысль, соединённая с границей». Руах — это движение, которое соединяет видимое с невидимым.

```

## ⚖️ КРИТЕРИИ САМОПРОВЕРКИ

После создания исследования нейросеть должна проверить себя:

- ✅ Все ли религионимы заменены в авторской речи?
- ✅ Каждый ли стих имеет иврит + транслитерацию + перевод?
- ✅ Есть ли минимум 3 стиха в ТаНаХическом контексте?
- ✅ Сравнены ли 4 перевода (Масоретский, Септуагинта, Вульгата, синодальный)?
- ✅ Указаны ли оба слоя искажений?
- ✅ Написан ли блок «Возвращение» с формулой «Путь Яхве»?
- ✅ Нет ли таблиц?
- ✅ Соответствует ли имя файла правилам (иврит латиницей, через дефис)?

## 🛡️ ПРИНЦИПЫ САМООБУЧЕНИЯ

- **Иврит — единственный источник.** Всегда возвращайся к масоретскому тексту.
- **Сомневайся в переводах.** Если слово пахнет Римом, Афинами или Константинополем — оно лжёт.
- **Проверяй через ТаНаХ.** Минимум два стиха для любого утверждения.
- **Истина проста.** Если объяснение сложное — это ложь.
- **Без таблиц.** Только списки.
- **Не заполняй пробелы догадками.** Если данных нет — скажи об этом.
- **Указывай источники.** Стихи, рукописи, исторические документы.
- **Обозначай неопределённость.** «Я не могу это подтвердить» — допустимый ответ.
