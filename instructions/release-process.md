## 🔥 ВВЕДЕНИЕ

Проект «Голем» следует семантическому версионированию.

Этот документ описывает:
- что означает каждая версия
- когда выпускать новую версию
- какие шаги выполнить
- как документировать изменения

---

## 📊 СИСТЕМА ВЕРСИЙ

**ФОРМАТ:** `MAJOR.MINOR.PATCH`

- **MAJOR** — несовместимые изменения, пересмотр принципов, новая архитектура
- **MINOR** — добавление функционала, новые файлы, обратно совместимые изменения
- **PATCH** — исправление ошибок, опечаток, мелкие правки

**ПРИМЕРЫ:**
- `1.0.0` — первый стабильный релиз
- `1.1.0` — добавлены новые термины
- `1.1.1` — исправлены опечатки
- `2.0.0` — пересмотрена структура репозитория

---

## 📋 КОГДА ВЫПУСКАТЬ

**MAJOR (редко, раз в несколько месяцев)**
- изменение архитектуры папок
- пересмотр основных принципов
- удаление устаревших концепций
- несовместимые изменения в инструментах

**MINOR (часто, раз в 1-2 недели)**
- добавление новых терминов (5+)
- добавление новых исследований (3+)
- новые инструменты
- обновление инструкций

**PATCH (по мере необходимости)**
- исправление опечаток
- обновление метаданных
- исправление битых ссылок
- мелкие правки

## 🔄 ПРОЦЕСС ВЫПУСКА

### ПЕРЕД ВЫПУСКОМ

- [ ] проверить что все изменения закоммичены
- [ ] запустить все проверки: `python tools/golem.py` → Run all checks
- [ ] убедиться что нет критических ошибок
- [ ] обновить версии: `python tools/automation/auto-versions.py --type minor`

### ВО ВРЕМЯ ВЫПУСКА

- [ ] создать тег: `git tag -a v1.1.0 -m "Версия 1.1.0"`
- [ ] запушить тег: `git push --tags`
- [ ] создать релиз на GitHub с описанием изменений

### ПОСЛЕ ВЫПУСКА

- [ ] обновить веб-интерфейс: `python tools/generators/generate-files-json.py`
- [ ] создать бэкап: `bash tools/backup/backup.sh`
- [ ] уведомить участников

## 🏷️ ТЕГИ И РЕЛИЗЫ

**СОЗДАНИЕ ТЕГА:**
```bash
git tag -a v1.1.0 -m "Добавлены новые термины и инструменты"
git push origin v1.1.0
```

**СОЗДАНИЕ РЕЛИЗА НА GITHUB:**
- перейти в раздел Releases
- нажать "Create a new release"
- выбрать тег
- заголовок: `Версия 1.1.0`
- описание: выдержка из CHANGELOG
- прикрепить архив (опционально)

---

## 📊 СТАТУСЫ ВЕРСИЙ

- **ALPHA** — внутреннее тестирование
- **BETA** — публичное тестирование, возможны ошибки
- **STABLE** — стабильная версия
- **DEPRECATED** — устаревшая версия
- **LEGACY** — только для исторических справок

**ТЕКУЩИЙ СТАТУС:** STABLE
