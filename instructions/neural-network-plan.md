## 🔥 ВИДЕНИЕ

Нейросеть «Эд» — это искусственный Свидетель. Машина, обученная методологии разоблачения, принципам перевода и корневой системе иврита. Она берёт любой стих из любого перевода, находит ивритский оригинал, разбирает до корня, сверяет с палео-ивритом, сравнивает переводы и выдаёт: что искажено, кем, когда, зачем и как правильно.

Конечная цель: достоверный перевод ТаНаХа на все языки мира, очищенный от религиозных, философских и культурных искажений.

## 📜 АРХИТЕКТУРА

### Модуль 1. Сканер текста

**Задача:** получить стих на любом языке, определить источник перевода, найти соответствующий стих в масоретском тексте.

**Вход:** текст на русском, английском, греческом, латыни.
**Выход:** стих масоретского текста + синодальный/другие переводы.

**Необходимые данные:**
- Параллельный корпус: масоретский текст — Септуагинта — Вульгата — синодальный — английский (KJV, NIV) — современные переводы
- Выравнивание по стихам (уже существует: Sefaria, BibleHub, Mechon-Mamre)

**Технологии:**
- Sentence embeddings для поиска соответствий
- TF-IDF + cosine similarity для выравнивания
- API Sefaria для получения масоретского текста

---

### Модуль 2. Разбор до корня

**Задача:** взять ивритское слово, определить трёхбуквенный корень, найти все однокоренные слова в ТаНаХе.

**Вход:** слово на иврите.
**Выход:** корень, однокоренные слова, частотность, контексты употребления.

**Необходимые данные:**
- Полная база трёхбуквенных корней иврита (≈1500–2000 корней)
- Словарь BDB (Brown-Driver-Briggs) в машиночитаемом виде
- Конкорданс Стронга (ивритские номера)
- Список всех словоформ с привязкой к корням

**Технологии:**
- Морфологический анализатор иврита (существуют открытые: HebMorph, ML-based)
- Граф корневых связей (networkx)
- Fine-tuned LLM на задачу определения корня

---

### Модуль 3. Сверка с палео-ивритом

**Задача:** разложить слово на буквы палео-иврита, выдать образ каждой буквы, собрать целостную картину.

**Вход:** слово на иврите.
**Выход:** цепочка букв с образами + интерпретация.

**Необходимые данные:**
- Таблица соответствия: квадратная буква → палео-ивритский символ → образ → значение
- База интерпретаций цепочек (из наших файлов)

**Технологии:**
- Lookup-таблица (детерминированный алгоритм)
- LLM для интерпретации цепочки образов

---

### Модуль 4. Сравнение переводов

**Задача:** взять стих, показать его в четырёх версиях, отметить расхождения, определить момент сдвига.

**Вход:** стих масоретского текста.
**Выход:** таблица расхождений + указание, где искажение + его причина.

**Необходимые данные:**
- Параллельный корпус (см. Модуль 1)
- База известных искажений (из наших файлов)
- Исторические даты переводов

**Технологии:**
- Diff-алгоритмы (пословное сравнение)
- NER для выделения терминов
- Классификатор типов искажений (fine-tuned на наших файлах)

---

### Модуль 5. Карта запрещённых слов

**Задача:** просканировать текст перевода, найти все религионимы, грецизмы, латинизмы, церковнославянизмы, выдать замену.

**Вход:** текст перевода.
**Выход:** лог проверки (❌⚠️✅) + список замен.

**Необходимые данные:**
- Полная карта запрещённых слов (`forbidden-words.md`)
- Контекстные правила (где допустимо, где нет)

**Технологии:**
- Поиск по словарю (детерминированный)
- Контекстный анализатор (LLM) для пограничных случаев

---

### Модуль 6. Обличение и возвращение

**Задача:** на основе найденных искажений сгенерировать обличение (кто, когда, зачем) и возвращение (правильный перевод + смысл).

**Вход:** результаты Модулей 2–5.
**Выход:** текст в формате нашего шаблона исследования.

**Технологии:**
- Fine-tuned LLM на корпусе наших файлов
- Few-shot prompting с примерами из `terminology/` и `researches/`

- **Параллельный корпус.** Масоретский текст — Септуагинта — Вульгата — синодальный — английский. Выровненный по стихам. Можно получить через Sefaria API.
- **Fine-tuning датасет.** Пары «стих перевода → наше исследование» для обучения LLM.

## 🔄 ЭТАПЫ РАЗРАБОТКИ

### Этап 1. Данные (1–2 месяца)

- Собрать базу корней иврита (из BDB и Стронга) в структурированном виде (JSON)
- Собрать параллельный корпус (5 переводов, выровненных по стихам)
- Оцифровать карту запрещённых слов в машиночитаемый формат
- Собрать fine-tuning датасет из наших файлов (пары: стих + разбор)

### Этап 2. Базовые модули (2–3 месяца)

- Модуль сканера текста (выравнивание стихов)
- Модуль разбора до корня (морфология + lookup)
- Модуль карты запрещённых слов (детерминированный)
- Модуль сверки с палео-ивритом (lookup)

### Этап 3. Интеллектуальные модули (3–4 месяца)

- Модуль сравнения переводов (diff + классификатор)
- Модуль обличения и возвращения (fine-tuned LLM)
- Интеграция всех модулей в пайплайн

### Этап 4. Тестирование и дообучение (2–3 месяца)

- Тест на 100 стихах из разных книг ТаНаХа
- Ручная проверка результатов
- Дообучение на ошибках
- Добавление новых типов искажений

### Этап 5. Масштабирование (3–6 месяцев)

- Поддержка всех языков (через multilingual LLM)
- Веб-интерфейс: вводишь стих — получаешь разбор
- API для разработчиков
- Telegram-бот: @EdWitnessBot
