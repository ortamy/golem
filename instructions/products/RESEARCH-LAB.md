## 🔥 ОПИСАНИЕ ПРОДУКТА

Исследовательская лаборатория (Research Lab) — это **web-приложение**, которое объединяет все инструменты, необходимые для работы с ТаНаХом, корнями, переводами и текстами, в едином интерфейсе.

Это не просто «редактор» или «чат». Это **рабочее место исследователя**, где можно:

- Открывать и редактировать файлы.
- Сравнивать переводы.
- Проверять текст на религионизмы.
- Общаться с нейросетью в контексте текущего файла.
- Хранить и искать корни, термины, палео-образы.

---

## 🧩 ФУНКЦИОНАЛЬНЫЕ МОДУЛИ

### 1. Текстовый редактор
- Открытие, редактирование и сохранение `.md`-файлов.
- Подсветка синтаксиса: метаданные, иврит, списки.
- Создание новых файлов по шаблону (TEMPLATE-RESEARCH, TEMPLATE-PERSON и др.).
- Автосохранение и версионирование (история изменений).

### 2. Чат с нейросетью
- Встроенный чат-интерфейс с ИИ-ассистентом.
- Контекст: нейросеть видит открытый файл, текущую папку, историю запросов.
- Возможность задавать вопросы по корням, подменам, переводам.
- Экспорт диалога в `.md` или `.txt`.

### 3. Библиотека корней и терминов
- Быстрый доступ к 150 корням, палео-образам, религионимам.
- Поиск по корню, по ивриту, по русскому слову.
- Фильтрация по категориям (творение, человек, движение, речь, отношения).

### 4. Сравнение переводов
- Масоретский текст (ТМ), Септуагинта (LXX), Пешитта, Синодальный.
- Отображение параллельных строк (иврит / транслит / перевод).
- Подсветка расхождений между версиями.

### 5. Чекер религионимов
- Проверка любого текста на религионизмы (по `data/religious-dictionary.json`).
- Подсветка найденных слов.
- Предложение замены с ивритским корнем.

### 6. Генератор исследовательских досок
- Создание визуальных досок из улик, заголовка и вывода.
- Экспорт в PNG, PDF, TXT.
- Генерация промпта для нейросети.

## 🛠 ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

- **Фреймворк:** не требуется (чистый HTML/JS/CSS).
- **Зависимости:** `html2canvas` для экспорта досок, `marked` для просмотра MD.
- **API:** для чата с нейросетью — подключение через OpenRouter или локальный сервер.
- **Хостинг:** статический (GitHub Pages) или с бэкендом (если понадобится сохранение файлов).

## 📌 ПРИНЦИП РАЗРАБОТКИ

- **Поэтапно:** сначала базовая структура, затем модули по одному.
- **Интеграция:** каждый модуль должен уметь работать с данными `content/` и `data/`.
- **Интерфейс:** все инструменты доступны через навигационную панель.
- **Адаптивность:** работает на десктопе, планшете и телефоне.
