## ЭТАП 2: ТЕХНИЧЕСКИЙ ДОЛГ

### 2.1 Прочитать
- `docs/TECHNICAL-DEBT.md`

### 2.2 Взять задачи с маркером `- [ ]`
- Сортировать: сначала те что можно сделать быстро (один файл, простая замена)
- Доклад: какие задачи берёт в работу

### 2.3 Выполнить задачи
- По одной
- После каждой: доклад что сделано

### 2.4 Отметить выполненные
- Заменить `- [ ]` на `- [x]` в TECH-DEBT.md

- Что осталось
- Что требует внимания человека

### 5.2 Обновить CONTROL.md
- Отметить выполненные задачи дня
- Добавить новые на завтра

### 5.3 Обновить BACKLOG.md

- Добавить 3-5 самых приоритетных на завтра
