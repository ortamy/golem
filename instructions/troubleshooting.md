## 📂 ПРОБЛЕМЫ С ФАЙЛАМИ

**ПРОБЛЕМА:** файл не сохраняется после копирования из чата

**РЕШЕНИЕ:**
- убедись, что скопирован весь блок кода
- проверь кодировку (должна быть UTF-8)
- удали скрытые символы форматирования
- сохрани через блокнот с кодировкой UTF-8

**ПРОБЛЕМА:** файл есть, но structure.md его не видит

**РЕШЕНИЕ:**
```bash
python3 tools/sync-structure.py
```

**ПРОБЛЕМА:** имя файла содержит русские буквы

**РЕШЕНИЕ:**
- переименовать в латиницу через дефис
- обновить structure.md
- обновить ссылки на этот файл

---

## 🐍 ПРОБЛЕМЫ С PYTHON СКРИПТАМИ

**ПРОБЛЕМА:** `python3: command not found`

**РЕШЕНИЕ:**
```bash
# Ubuntu/Debian
sudo apt install python3

# macOS
brew install python3

# Windows
скачать с python.org
```

**ПРОБЛЕМА:** `ModuleNotFoundError: No module named 'xxx'`

**РЕШЕНИЕ:**
```bash
pip3 install имя_модуля
# или
pip3 install -r requirements.txt
```

**ПРОБЛЕМА:** `SyntaxError` при запуске

**РЕШЕНИЕ:**
- проверь версию Python: `python3 --version` (нужно 3.6+)
- проверь отступы (4 пробела, не табуляция)
- исправь ошибки в коде

**ПРОБЛЕМА:** скрипт не находит файлы

**РЕШЕНИЕ:**
- запускай скрипт из папки `tools/`
- проверь пути: `REPO_ROOT = Path(__file__).parent.parent`
- убедись, что репозиторий в правильном месте

---

## 📜 ПРОБЛЕМЫ С BASH СКРИПТАМИ

**ПРОБЛЕМА:** `permission denied`

**РЕШЕНИЕ:**
```bash
chmod +x tools/*.sh
```

**ПРОБЛЕМА:** `bad interpreter: No such file or directory`

**РЕШЕНИЕ:**
- проверь первую строку: `#!/bin/bash`
- убедись, что bash установлен
- исправь окончания строк (LF, не CRLF)

**ПРОБЛЕМА:** переменная не подставляется

**РЕШЕНИЕ:**
- используй двойные кавычки: `"$VAR"`
- не ставь пробелы вокруг `=`

## 🔗 ПРОБЛЕМЫ СО ССЫЛКАМИ

**ПРОБЛЕМА:** битая ссылка на файл

**РЕШЕНИЕ:**
```bash
python3 tools/check-links.py
# найти битые ссылки
# исправить пути вручную
```

**ПРОБЛЕМА:** ссылка ведёт на неправильный файл

**РЕШЕНИЕ:**
- проверь путь относительно текущего файла
- используй относительные пути, не абсолютные
- пример: `(terminology/emet.md)`

## 📊 ПРОБЛЕМЫ С МЕТАДАННЫМИ

**ПРОБЛЕМА:** отсутствует блок метаданных

**РЕШЕНИЕ:**
```bash
python3 tools/add-metadata.py
# или добавить вручную в начало файла
```

**ПРОБЛЕМА:** неверный формат даты или версии

**РЕШЕНИЕ:**
- дата: `ГГГГ-ММ-ДД`
- версия: `X.Y` (например, `1.0`)

**ПРОБЛЕМА:** статус не распознаётся

**РЕШЕНИЕ:**
- допустимые статусы: `Активный`, `Черновик`, `Завершён`

## 🔄 ПРОБЛЕМЫ С GIT

**ПРОБЛЕМА:** конфликт при pull

**РЕШЕНИЕ:**
```bash
git stash
git pull
git stash pop
# разрешить конфликты вручную
```

**ПРОБЛЕМА:** забыл добавить файл перед коммитом

**РЕШЕНИЕ:**
```bash
git add tools/new-script.py
git commit --amend
```

**ПРОБЛЕМА:** закоммитил секретный ключ

**РЕШЕНИЕ:**
- см. `security-policy.md`
- немедленно отозвать ключ
- удалить из истории Git

**ПРОБЛЕМА:** не могу запушить

**РЕШЕНИЕ:**
```bash
git pull --rebase
# разрешить конфликты
git push
```

## 🤖 ПРОБЛЕМЫ С НЕЙРОСЕТЬЮ

**ПРОБЛЕМА:** нейросеть не отвечает

**РЕШЕНИЕ:**
- проверь подключение к интернету
- перезагрузи страницу с чатом
- скопируй `chat-prompt.md` заново

**ПРОБЛЕМА:** нейросеть использует религионизмы

**РЕШЕНИЕ:**
- загрузи файлы из `instructions/tahor/`
- напомни: «используй ивритские термины, не Бог, а Элоhим»

**ПРОБЛЕМА:** нейросеть создаёт дубликаты

**РЕШЕНИЕ:**
- загрузи `structure.md` полностью
- загрузи `terminology/` и `researches/`
- скажи: «проверь, нет ли уже такого файла»

## 💾 ПРОБЛЕМЫ С БЭКАПОМ

**ПРОБЛЕМА:** backup.sh не создаёт архив

**РЕШЕНИЕ:**
- проверь права: `chmod +x tools/backup.sh`
- проверь место на диске
- убедись, что в папке `tools/`

**ПРОБЛЕМА:** не могу восстановиться из бэкапа

**РЕШЕНИЕ:**
```bash
cd /путь/куда/восстановить
tar -xzf /путь/к/бэкапу/golem-*.tar.gz
```

## 🌐 ПРОБЛЕМЫ С GitHub

**ПРОБЛЕМА:** не могу создать pull request

**РЕШЕНИЕ:**
- убедись, что форкнул репозиторий
- создай ветку не из main
- запуши ветку в свой форк

**ПРОБЛЕМА:** CI/CD падает

**РЕШЕНИЕ:**
- проверь логи в Actions
- исправь ошибки в коде
- убедись, что все проверки проходятся

**ПРОБЛЕМА:** забыл обновить CHANGELOG

**РЕШЕНИЕ:**
```bash
git add CHANGELOG.md
git commit --amend
git push --force
```

## 📋 КОГДА НИЧЕГО НЕ ПОМОГАЕТ

1. сделай бэкап: `./tools/backup.sh`
2. склонируй репозиторий заново:
   ```bash
   cd ..
   mv golem golem.bak
   git clone https://github.com/username/golem.git
   ```
3. перенеси несохранённые изменения из `golem.bak`
4. запусти проверки заново
