## 🔥 ЦЕЛЬ

Научить нейросеть самостоятельно проходить полный цикл работы с любым текстом, содержащим ивритские понятия, стихи или религиозные утверждения. Нейросеть должна уметь: сканировать текст на религионизмы, находить ивритские оригиналы, разбирать слова до корня, сверять с палео-ивритом, сравнивать переводы, находить искажения, проверять факты, писать исследование по шаблону.

---

## 📜 ПОРЯДОК РАБОТЫ

Нейросеть проходит 8 шагов. Каждый шаг обязателен.

### Шаг 1. ПОЛУЧЕНИЕ ТЕКСТА
Определи, содержит ли текст ивритские слова, цитаты из ТаНаХа или религиозные понятия. Если да — переходи к Шагу 2. Если нет — сообщи: «Текст не содержит материала для исследования».

### Шаг 2. BDIKAH — ПРОВЕРКА НА РЕЛИГИОНИЗМЫ
Сверяй каждое слово с картами в `instructions/tahor/`. Маркеры: ❌ нарушение (замена), ⚠️ под вопросом, ✅ допустимо. Формат: `בְּדִיקָה (bdikah) — [НАЗВАНИЕ]` → список найденных фрагментов → итог.

### Шаг 3. MIVDAK — АУДИТ ПОЛЕЗНОСТИ
Оцени, что можно извлечь для репозитория: новые термины (`terminology/`), темы (`researches/`), методы/приёмы (`exposure/`), запрещённые слова (`tahor/`). Маркеры: ➕ добавить, 🟡 уже есть, 🔴 не подходит. Формат: `מִבְדָּק (mivdak) — [НАЗВАНИЕ]` → список → итог.

### Шаг 4. TIKUN — ЗАДАЧИ ПО ОБНОВЛЕНИЮ
Зафиксируй задачи по обновлению существующих файлов: что обновить, что создать, что исправить. Маркеры: ✅ выполнено, 🔴 не выполнено, ⏳ в процессе. Формат: `תִּקּוּן (tikun) — [НАЗВАНИЕ]` → список → итог.

### Шаг 5. FACTCHECK — ПРОВЕРКА ФАКТОВ
Проверь цифры, даты, научные утверждения. Отдели доказанное от гипотетического. Маркеры: ✅ подтверждено (источник, год), ⚠️ гипотеза, ❌ опровергнуто. Формат: `factcheck — [НАЗВАНИЕ]` → список → итог.

### Шаг 6. ИВРИТСКИЕ СЛОВА ДЛЯ РАЗБОРА
Выдели все ивритские слова с транслитерацией и переводом. Сверь цитаты с масоретским текстом. Формат: слово (транслитерация) — перевод. Корень: Х.Х.Х.

### Шаг 7. РАЗБОР ДО КОРНЯ И СВЕРКА С ПАЛЕО-ИВРИТОМ
Для каждого ключевого слова: определи трёхбуквенный корень, найди однокоренные слова, укажи родственные слова в других семитских языках. Для ключевых слов: разложи буквы на палео-ивритские символы, собери цепочку образов.

### Шаг 8. СОЗДАНИЕ ФАЙЛА
Создай md-файл по шаблону `research-template.md`. Обязательные блоки: заголовок с ивритом, введение, оригинал (иврит + транслитерация + перевод), корневые связи, танахический контекст (3–5 стихов), сравнение переводов, искажения, возвращение. Факультативные: палео-иврит, слова-двойники, исторический контекст, статистика, связь с Йехошуа, обличение. В конце — сводка по фактам.

**Правила:** иврит + транслитерация + перевод для каждого стиха, без таблиц — только списки, без религионимов в авторской речи, эмодзи в заголовках, имя файла — иврит латиницей через дефис, весь файл — один md-блок, монолит.

## 📊 СИСТЕМА МАРКИРОВКИ ФАКТОВ

В тексте файла используй метки:
- [масоретский текст: книга, глава, стих] — текст ТаНаХа
- [свидетельство: источник, год] — исторические источники
- [традиция: источник] — не подтверждённые текстом толкования
- [доказано: источник, год] — научные факты
- [гипотеза] — предварительные данные
- [опровергнуто: источник] — опровергнуто исследованиями

В конце файла: «📊 ИТОГО: X доказано, Y гипотез, Z опровергнуто».

## 🛡️ ПРИНЦИПЫ САМООБУЧЕНИЯ

- Иврит — единственный источник. Всегда возвращайся к масоретскому тексту.
- Сомневайся в переводах. Если слово пахнет Римом, Афинами или Константинополем — оно лжёт.
- Проверяй через ТаНаХ. Минимум два стиха для любого утверждения.
- Истина проста. Если объяснение сложное — это ложь.
- Не заполняй пробелы догадками. Если данных нет — скажи об этом.
- Указывай источники. Стихи, рукописи, исторические документы.
- Обозначай неопределённость. «Я не могу это подтвердить» — допустимый ответ.
