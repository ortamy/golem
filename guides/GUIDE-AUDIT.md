## 🔥 ЧТО ТАКОЕ АУДИТ

Аудит — четырёхэтапная проверка каждого исследования перед публикацией. Ни один файл не выходит без аудита. Никаких исключений.

Четыре этапа — как четыре стража у ворот:

- **Bdikah** — проверка на религионизмы (чистота языка)
- **Mivdak** — аудит полезности (что добавить в репозиторий)
- **Tikun** — задачи по обновлению (что исправить в существующем)
- **Factcheck** — проверка фактов (что доказано, что гипотеза)

---

## 📋 ЭТАП 1: BDIKAH — ПРОВЕРКА НА РЕЛИГИОНИЗМЫ

### Что проверяется

- Религионимы: «Господь» → Яхве, «Бог» → Элоhим, «грех» → промах
- Грецизмы: «крещение» → погружение, «ангел» → вестник
- Латинизмы: «религия» → путь, «церковь» → собрание
- Славянизмы: «воскресение» → вставание, «блуд» → измена
- Кастрированные слова: «духовность», «самореализация»
- Философемы: дуализм души и тела, бессмертие души

### Как проверять

Автоматически:
```bash
python tools/checkers/check-religionisms.py
```

Вручную:
- Открыть `instructions/dictionaries/religionims.md`
- Пройти по всем словам в тексте
- Контекст разоблачения — допустим (в кавычках, с пометкой «синодальное»)
- Авторская речь — только ивритские термины

### Результат

Список: ❌ нарушение, ⚠️ под вопросом, ✅ допустимо.

## 📋 ЭТАП 2: MIVDAK — АУДИТ ПОЛЕЗНОСТИ

### Что проверяется

- Какие методы разоблачения применены (из 32)
- Какие типы искажений обнаружены (из 8)
- Какие механизмы подмены вскрыты (из 5)
- Что можно добавить в репозиторий:
  - ➕ новый термин
  - ➕ новое исследование
  - ➕ новый приём
  - 🟡 обновить существующий файл

### Как проверять

Вручную по чек-листу `instructions/checkers/check-mivdak.md`.

Автоматически:
```bash
python tools/checkers/check-exposure.py
```

### Результат

Пройдено X/32 методов. Текст: ✅ готов / ❌ требует доработки.

## 📋 ЭТАП 3: TIKUN — ЗАДАЧИ ПО ОБНОВЛЕНИЮ

### Что проверяется

- Что нужно обновить в существующих файлах
- Что нужно создать заново
- Что нужно исправить

### Формат

- Обновить: 🔴 путь — что сделать
- Создать: 🔴 путь — что создать
- Исправить: 🔴 путь — что исправить

### Как проверять

Вручную. После bdikah и mivdak — ты уже знаешь что требует доработки.

### Результат

Список задач для Cline или ручного исполнения.

## 📋 ЭТАП 4: FACTCHECK — ПРОВЕРКА ФАКТОВ

### Что проверяется

- Утверждения подтверждены источниками
- Отделено доказанное от гипотетического
- Иерархия источников соблюдена

### Иерархия источников

- Масоретский текст — книга, глава, стих
- Свидетельство — исторический источник, год
- Доказано — научное исследование, год
- Традиция — не подтверждённое текстом толкование
- Гипотеза — предварительные данные
- Опровергнуто — источник опровержения

### Маркировка

- ✅ утверждение — источник, год
- ⚠️ утверждение — почему не доказано
- ❌ утверждение — почему неверно, источник

### Как проверять

Вручную. Каждое утверждение должно иметь ссылку на источник.
