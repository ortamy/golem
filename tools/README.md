## 🔧 1. SYNC-STRUCTURE.PY

### НАЗНАЧЕНИЕ
Автоматически обновляет файл `structure.md` в корне репозитория, синхронизируя его с реальной файловой системой.

### КОГДА ИСПОЛЬЗОВАТЬ
- после добавления новых файлов или папок
- после переименования или удаления файлов
- перед коммитом, чтобы структура была актуальной
- для проверки, что все файлы отражены в документации

### КАК РАБОТАЕТ
- сканирует репозиторий, исключая `.git`, `tools`, `drafts`, `ideas`
- генерирует новое содержимое `structure.md`
- сохраняет с обновлённой датой и версией

### ЗАПУСК
```bash
cd /путь/к/golem/tools
python3 sync-structure.py
```

### ПРИМЕР ВЫВОДА
```
🔄 Синхронизация structure.md...
✅ Обновлено: /path/to/golem/structure.md
📝 Проверьте изменения и сделайте git commit
```

### ПОСЛЕ ЗАПУСКА
- проверь изменения в `structure.md` через `git diff`
- если всё правильно, сделай коммит:
```bash
git add structure.md
git commit -m "Обновление структуры"
```

## 📝 2. ADD-METADATA.PY

### НАЗНАЧЕНИЕ
Массово добавляет блоки метаданных во все md-файлы в папках `terminology/` и `researches/`.

### КОГДА ИСПОЛЬЗОВАТЬ
- для новых файлов, созданных без метаданных
- для приведения старых файлов к единому стандарту
- после изменения формата метаданных

### КАК РАБОТАЕТ
- проверяет, есть ли уже метаданные в файле
- если нет — добавляет их после заголовка
- извлекает тему из первого заголовка или имени файла

### ФОРМАТ МЕТАДАННЫХ
```markdown
**Метаданные файла**
- **Файл:** `terminology/emet.md`
- **Версия:** 1.0
- **Дата создания:** 2026-06-05
- **Последнее обновление:** 2026-06-05
- **Причина обновления:** Создание файла
- **Статус:** Активный
- **Тема:** Истина
```

### ЗАПУСК
```bash
cd /путь/к/golem/tools
python3 add-metadata.py
```

### ПРИМЕР ВЫВОДА
```
📝 Добавление метаданных в файлы...
  ✅ Добавлены метаданные: emet.md
  ✅ Добавлены метаданные: chesed.md
  ⏭️ Пропуск (уже есть метаданные): yhwh.md

✅ Обработано файлов: 45
```

### ВАЖНО
- скрипт не перезаписывает существующие метаданные
- перед массовым запуском протестируй на одном файле
- после запуска проверь несколько файлов вручную

## 🔍 3. CHECK-NAMING.PY

### НАЗНАЧЕНИЕ
Проверяет имена всех md-файлов на соответствие правилам именования.

### КОГДА ИСПОЛЬЗОВАТЬ
- перед коммитом для проверки новых файлов
- для аудита репозитория
- для выявления нарушений перед исправлением

### ПРАВИЛА ИМЕНОВАНИЯ
- только латиница (a-z)
- дефис между словами
- расширение `.md`
- нижний регистр
- максимум 50 символов
- без русских букв
- без двойных дефисов
- без дефиса перед .md

### ЗАПУСК
```bash
cd /путь/к/golem/tools
python3 check-naming.py
```

### ПРИМЕР ВЫВОДА (ЕСЛИ ЕСТЬ НАРУШЕНИЯ)
```
❌ НАРУШЕНИЯ:

  📄 researches/My-File.md
     • Неверный формат (только a-z и дефис): My-File.md
     • Заглавные буквы: My-File.md

  📄 terminology/неверное_имя.md
     • Русские буквы: неверное_имя.md
     • Неверный формат (только a-z и дефис): неверное_имя.md

📊 ИТОГО: 2 файлов с нарушениями из 150
```

### ПРИМЕР ВЫВОДА (ВСЁ ХОРОШО)
```
✅ Все 150 файлов названы корректно
```

### ЧТО ДЕЛАТЬ С НАРУШЕНИЯМИ
- переименовать файлы вручную
- или использовать скрипт для массового переименования (пока не реализован)

## 📦 4. EXPORT-REPO.SH

### НАЗНАЧЕНИЕ
Собирает содержимое всех md-файлов репозитория в один текстовый файл `export.txt`.

### КОГДА ИСПОЛЬЗОВАТЬ
- для загрузки всех файлов в нейросеть (DeepSeek, ChatGPT)
- для поиска по всему репозиторию
- для создания бэкапа в читаемом формате
- для анализа всех файлов сразу

### КАК РАБОТАЕТ
- находит все `.md` файлы в репозитории
- для каждого записывает путь и содержимое
- разделитель: `=== путь/к/файлу.md ===`

### ЗАПУСК
```bash
cd /путь/к/golem/tools
chmod +x export-repo.sh
./export-repo.sh
```

### ПРИМЕР ВЫВОДА
```
📁 ВЫГРУЗКА РЕПОЗИТОРИЯ
========================
✅ Готово: /path/to/golem/export.txt
📊 Объём: 15420 строк
📁 Размер: 2.3M
```

### ИСПОЛЬЗОВАНИЕ export.txt
- открыть в любом текстовом редакторе
- отправить в нейросеть как контекст
- искать по всем файлам сразу: `grep "слово" export.txt`

### ПРИМЕЧАНИЯ
- файл создаётся в корне репозитория (рядом с README.md)
- исключает сам export.txt из выгрузки
- при повторном запуске перезаписывает файл

## 💾 5. BACKUP.SH

### НАЗНАЧЕНИЕ
Создаёт сжатый бэкап всего репозитория (исключая `.git`).

### КОГДА ИСПОЛЬЗОВАТЬ
- перед крупными изменениями
- ежедневно для автоматического бэкапа
- перед экспериментальными скриптами

### КАК РАБОТАЕТ
- создаёт папку `../golem-backups` (рядом с репозиторием)
- формирует архив с датой и временем в имени
- удаляет старые бэкапы, оставляя только 10 последних

### ЗАПУСК
```bash
cd /путь/к/golem/tools
chmod +x backup.sh
./backup.sh
```

### ПРИМЕР ВЫВОДА
```
💾 БЭКАП РЕПОЗИТОРИЯ
====================
Источник: /path/to/golem
Назначение: /path/to/golem-backups/golem-2026-06-05_14-30-00.tar.gz

✅ Бэкап создан: /path/to/golem-backups/golem-2026-06-05_14-30-00.tar.gz
📊 Размер: 1.8M
🧹 Очищено: оставлено 10 последних бэкапов
```

### ВОССТАНОВЛЕНИЕ ИЗ БЭКАПА
```bash
cd /путь/куда/восстановить
tar -xzf /путь/к/golem-backups/golem-2026-06-05_14-30-00.tar.gz
```

### АВТОМАТИЧЕСКИЙ БЭКАП (CRON)
Добавь в crontab для ежедневного бэкапа:
```bash
crontab -e
# Добавить строку:
0 2 * * * /путь/к/golem/tools/backup.sh
```

## 🛠️ УСТРАНЕНИЕ ПРОБЛЕМ

### ОШИБКА: permission denied
```bash
chmod +x tools/*.sh
chmod +x tools/*.py
```

### ОШИБКА: python3 not found
Установи Python 3:
```bash
# Ubuntu/Debian
sudo apt install python3

# macOS
brew install python3
```

### ОШИБКА: Нет прав на запись
Убедись, что ты владелец репозитория:
```bash
sudo chown -R $USER:$USER /путь/к/golem
```
