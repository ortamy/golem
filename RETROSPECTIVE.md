## 🔍 РЕГУЛЯРНАЯ ПРОВЕРКА АРХИТЕКТУРЫ, СТРУКТУРЫ И ИМЕНОВАНИЯ ФАЙЛОВ

Каждая ретроспектива включает аудит по следующим пунктам.

### 1. ПРОВЕРКА АРХИТЕКТУРЫ

**Вопросы для проверки:**
- все ли папки соответствуют своему назначению?
- нет ли дублирующихся папок (например, `davar/` в корне и `drafts/davar-language.md`)?
- не пора ли создать новую папку (например, `methodology/`, `neural-network/`)?
- не устарела ли архитектура под текущие задачи?

**Архитектурные принципы:**
- корень — только основные файлы (README, структура, бэклог, роадмап, ретроспектива)
- `instructions/` — только для нейросети (критично для настройки)
- `ideas/` — идеи, планы, нереализованное
- `drafts/` — черновики, временное, техдолг
- `terminology/` — готовые термины
- `researches/` — готовые исследования
- `davar/` — язык программирования (когда будет реализация)

### 2. ПРОВЕРКА СТРУКТУРЫ

**Вопросы для проверки:**
- `structure.md` синхронизирован с реальной файловой системой?
- все ли файлы перечислены в `structure.md`?
- нет ли файлов, которые не отражены в структуре?
- нет ли в структуре файлов, которых физически нет?

**Процесс синхронизации:**
- запустить `tree /f > structure.txt` (Windows) или `find . -name "*.md" | sort` (Linux/Mac)
- сравнить с текущим `structure.md`
- обновить `structure.md` до новой версии
- зафиксировать изменения в CHANGELOG.md

### 3. ПРОВЕРКА ИМЕНОВАНИЯ ФАЙЛОВ

**Вопросы для проверки:**
- все ли файлы названы по правилу: иврит латиницей через дефис?
- нет ли файлов с русскими буквами?
- нет ли файлов без расширения `.md`?
- нет ли опечаток в именах?

**Правила именования:**
- только латиница (a-z)
- ивритские слова транслитерируются
- дефис между словами, не пробел
- расширение `.md`
- нижний регистр

**Примеры правильных имён:**
- `yhwh-name.md`
- `brit-hadashah.md`
- `exposure-methods.md`

**Исключения:**
- `README.md`, `CHANGELOG.md`, `BACKLOG.md` — английские, потому что стандарт GitHub

**Текущие нарушения:**
- `research-princilples.md` — опечатка, должно быть `research-principles.md`
- `SCTRUCTURE.md` — опечатка, должно быть `STRUCTURE.md` или удалить дубль
- `substitution-of-the-name` — нет расширения `.md`

### 4. ПРОВЕРКА МЕТАДАННЫХ В ФАЙЛАХ

**Вопросы для проверки:**
- у каждого файла есть блок метаданных?
- указана версия, дата создания, последнее обновление, статус, тема?
- версия обновляется при изменениях?
- причина обновления описана?

**Формат метаданных:**
```
**Метаданные файла**
- **Файл:** `путь/имя-файла.md`
- **Версия:** X.Y
- **Дата создания:** ГГГГ-ММ-ДД
- **Последнее обновление:** ГГГГ-ММ-ДД
- **Причина обновления:** краткое описание
- **Статус:** Активный / Черновик / Завершён
- **Тема:** одно предложение
```

### 5. ПРОВЕРКА ДУБЛИКАТОВ

**Вопросы для проверки:**
- нет ли двух файлов об одном и том же понятии?
- нет ли пересекающихся исследований?
- нет ли термина, который разобран и в `terminology/`, и в `researches/`?

**Как проверять:**
- использовать `find` и `grep` для поиска ключевых слов
- перед созданием нового файла проверять, нет ли уже существующего
- при обнаружении дубликата — объединить или удалить один

### 6. ПРОВЕРКА ЦЕЛОСТНОСТИ ССЫЛОК

**Вопросы для проверки:**
- все ли внутренние ссылки ведут на существующие файлы?
- нет ли битых ссылок в `README.md`, `structure.md`, `ROADMAP.md`?
- ссылки используют правильные пути (относительные или абсолютные)?
