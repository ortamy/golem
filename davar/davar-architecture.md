## 🔥 ВВЕДЕНИЕ

Давар (דָּבָר) — язык архитектуры творения. В иврите слово и дело — одно: דבר.

Это не очередной язык программирования. Это попытка вернуть языку его творческую силу.

Цель Давара — не решение алгоритмических задач, а выражение истины в структуре кода.

---

## 📜 ФУНДАМЕНТАЛЬНЫЕ ПРИНЦИПЫ

**1. СЛОВО РАВНО ДЕЛО**

В Даваре нет разрыва между объявлением и выполнением. Написав код, ты уже совершил действие.

**2. ВРЕМЯ — НЕ ЛИНИЯ**

Нет прошлого, настоящего и будущего. Есть сбывшееся (завершённое) и несбывшееся (незавершённое).

**3. КОНКРЕТИКА, НЕ АБСТРАКЦИЯ**

Нет абстрактных типов «объект» или «класс». Есть конкретные сущности: дом, путь, имя, союз.

**4. ВЕРНОСТЬ, А НЕ ФУНКЦИЯ**

Программа не «выполняет функции». Она хранит верность своим обязательствам.

**5. ОТДЕЛЁННОСТЬ, А НЕ ИЗОЛЯЦИЯ**

Код может быть отделён (קודש) для определённой цели, но не изолирован от системы.

## 🧱 КЛЮЧЕВЫЕ ПОНЯТИЯ

**דָּבָר (ДАВАР) — СЛОВО-ДЕЛО**

Минимальная единица программы. Объявляет намерение и немедленно его исполняет.

**בְּרִית (БРИТ) — СОЮЗ**

Связь между двумя даварим. Союз нельзя нарушить — только разорвать.

**דֶּרֶךְ (ДЭРЭХ) — ПУТЬ**

Последовательность даварим. Не «функция» и не «алгоритм». Путь можно пройти или свернуть.

**שֵׁם (ШЭМ) — ИМЯ**

Идентификатор, который раскрывает суть. Не произвольная метка.

**עֵד (ЭД) — СВИДЕТЕЛЬ**

Специальный давар, который проверяет истинность других даварим.

**מִצְוָה (МИЦВА) — СВЯЗЬ**

Правило, соединяющее действие с его последствием.

## 📐 СИНТАКСИС (КОНЦЕПЦИЯ)

**ОБЪЯВЛЕНИЕ ДАВАРА:**

```
דָּבָר имя_сущности [תֹּאַר (свойства)] {
    // тело
}
```

**ПРИМЕР:**

```
דָּבָר שַׁבָּת מִצְוָה {
    יְנוּחָה (покой)
    הַפְסָקָה (остановка)
}
```

**СОЗДАНИЕ СОЮЗА:**

```
בְּרִית союз {
    связывает: давар1, давар2
    на_условиях: [правило1, правило2]
}
```

**ОПРЕДЕЛЕНИЕ ПУТИ:**

```
דֶּרֶךְ имя_пути {
    צַעַד (шаг) 1: действие
    צַעַד (шаг) 2: действие
    ...
}
```

**СВИДЕТЕЛЬСТВО:**

```
עֵד проверка {
    если: условие
    то: действие
    иначе: действие
}
```

## 🎯 ТИПЫ ДАВАРИМ

- `קֹדֶשׁ` (отделённый) — для специальных целей
- `חֹל` (обычный) — повседневный
- `טָהוֹר` (пригодный) — безопасный для использования
- `טָמֵא` (непригодный) — требует очищения

## 📊 ПРИМЕР ПРОГРАММЫ

**HELLO WORLD НА ДАВАРЕ:**

```
דָּבָר שָׁלוֹם {
    אֱמוֹר "שָׁלוֹם עוֹלָם"
    אֱמוֹר "מִן הַדֶּרֶךְ"

דֶּרֶךְ הַתְחָלָה {
    צַעַד 1: קְרָא שָׁלוֹם

```

**ПРОВЕРКА ИСТИНЫ:**

```
עֵד אֱמֶת {

    אָז אֱמוֹר "אֱמֶת"
    אַחֶרֶת אֱמוֹר "שֶׁקֶר"
}
```

## 🔄 ОТЛИЧИЯ ОТ СУЩЕСТВУЮЩИХ ЯЗЫКОВ

**ЕДИНИЦА**
- В обычных языках: функция, метод
- В Даваре: давар (слово-дело)

**СВЯЗЬ**
- В обычных языках: вызов, наследование
- В Даваре: союз (брит)

**ПОСЛЕДОВАТЕЛЬНОСТЬ**
- В обычных языках: алгоритм
- В Даваре: путь (дэрех)

**ПРОВЕРКА**
- В обычных языках: assert, if
- В Даваре: свидетель (эд)

**ВРЕМЯ**
- В обычных языках: линейное
- В Даваре: сбывшееся/несбывшееся

**АБСТРАКЦИЯ**
- В обычных языках: классы, интерфейсы
- В Даваре: отделённость (кодеш)

## ⚠️ ОГРАНИЧЕНИЯ

- Давар не заменит C++, Python или Go для большинства задач
- скорость выполнения будет ниже
- экосистемы нет
- цель — не производительность, а выразительность истины
