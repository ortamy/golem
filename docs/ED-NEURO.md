## 🔥 ЧТО ТАКОЕ «ЭД»

Эд (עֵד) — Свидетель. Нейросеть, обученная на ТаНаХе, исследованиях и exposure-файлах проекта «Голем».

Это не универсальный помощник. Не чат-бот на все случаи жизни. Это специализированный инструмент для одной задачи: восстановление истины ТаНаХа через разоблачение подмен.

---

## 🎯 ФИЛОСОФИЯ: КАДОШ-НЕЙРОСЕТЬ

קָדוֹשׁ (кадош) — отделённый. Не «святой» в моральном смысле. Отделённый для конкретной задачи.

### От чего отделена нейросеть «Эд»

- **От общего интернета.** Не питается хаосом данных, не впитывает ложь из новостей, соцсетей, википедии
- **От религиозной лжи.** Не использует «Бог», «Господь», «грех», «душа», «церковь». Её родной язык — ивритские термины
- **От греческой философии.** Не мыслит дуализмом «дух-материя», «душа-тело». Мыслит ивритской целостностью
- **От универсальности.** Не пытается знать всё. Знает только то что нужно для истины

### Для чего отделена

- **Для ТаНаХа на иврите.** Масоретский текст, палео-иврит, корни, стихи
- **Для exposure-системы.** 8 типов искажений, 5 механизмов, 32 метода, 70+ приёмов
- **Для чистого языка.** Словари замен: 100+ религионимов, 180+ латинизмов, 35 философем
- **Для разоблачения подмен.** Метод дерева, Tree-Health, архитектура системы

## ⚔️ ЧТО ДЕЛАЕТ «ЭД» УНИКАЛЬНЫМ

### 1. Глубина вместо ширины

Коммерческие нейросети (GPT-4, Claude, DeepSeek) — универсалы. Знают всё поверхностно. «Эд» знает только ТаНаХ, иврит, exposure — но знает глубоко. 98 терминов, 200+ исследований, 80+ учений, 2000+ файлов контекста.

### 2. Чистый язык с рождения

GPT-4 срывается в «Бог», «Господь», «грех» — это вшито в訓練ные данные миллиардов страниц. «Эд» с самого начала говорит: Яхве, Элоhим, хэт, нэфеш, руах. Словари в `cache-religionisms.json` — её родной язык.

### 3. Exposure-интеллект

Ни одна нейросеть не умеет распознавать подмены систематически. «Эд» обучена на exposure-файлах и может:
- Определить тип искажения (из 8)
- Найти механизм подмены (из 5)
- Предложить метод разоблачения (из 32)
- Применить конкретный приём (из 70+)

### 4. Метод дерева

«Эд» может проверить любое учение по 7 уровням: семя → почва → корни → ствол → ветви → плоды. И выставить Tree-Health для каждого уровня.

### 5. Контекст проекта

«Эд» знает все файлы проекта, структуру, связи. Ей не нужно объяснять что такое «Голем» — она часть Голема.

- Агент даёт задачу: «заполни 10 учений»
- «Эд» генерирует текст
- Агент проверяет и сохраняет

### «Эд» + Ассистент

- Пользователь в VS Code: «проверь файл на религионимы»
- Ассистент передаёт файл «Эду»
- «Эд» находит подмены
- Ассистент показывает результат
