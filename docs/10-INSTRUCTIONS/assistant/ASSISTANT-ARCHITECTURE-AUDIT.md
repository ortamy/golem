---

### 🔍 4. Проверка целостности `.md`-файлов

Для каждого `.md`-файла в `content/` и `instructions/` проверить:

- [ ] Есть блок метаданных в начале (обязательные поля: `Файл`, `Версия`, `Статус`, `Тема`)
- [ ] Есть заголовок первого уровня (`#`)
- [ ] Есть хотя бы один раздел (`##` или `###`)
- [ ] Размер файла > 100 байт
- [ ] Нет шаблонных заглушек (`TODO`, `[ВСТАВИТЬ]`, `...`, `XXX`)
- [ ] Нет религионимов в авторской речи (использовать `data/religious-dictionary.json`)
- [ ] Ссылки на связанные файлы ведут на существующие файлы

**Если не выполнено — отметить как `НЕПОЛНЫЙ` или `ПОВРЕЖДЁН`.**

---

### 🔍 5. Проверка `ARCHITECTURE.md` и `STRUCTURE.md`

- [ ] `ARCHITECTURE.md` соответствует реальной структуре (нет папок, которых нет на диске)
- [ ] `STRUCTURE.md` содержит актуальный список всех файлов
- [ ] Нет расхождений между описанием и реальностью

---

### 🔍 6. Проверка дублей

Найти файлы с одинаковым названием (в разных папках) и файлы с одинаковым содержимым (даже если названия разные). Отметить в отчёте как `ДУБЛЬ`.

---

### 🔍 7. Проверка неиспользуемых файлов

Найти файлы, на которые нет ссылок ни из каких `.md`, `.html`, `.js`, `.py`, `.json`. Отметить в отчёте как `НЕ ИСПОЛЬЗУЕТСЯ`.

---

### 🔍 8. Проверка метаданных

Для всех `.md`-файлов с метаданными проверить:

- [ ] `Версия` — формат `X.Y` (например, `1.0`)
- [ ] `Дата создания` — корректная дата
- [ ] `Последнее обновление` — не старше 1 месяца (если файл изменялся)
- [ ] `Причина обновления` — не пустая, если файл изменялся
- [ ] `Хеш` — если есть, соответствует содержимому (можно проверить через `hashlib.md5`)

## ⚙️ КРИТЕРИИ ОЦЕНКИ

| Статус | Критерий |
|--------|----------|
| OK | Все проверки пройдены, расхождений нет |
| ТРЕБУЕТ ВНИМАНИЯ | Есть пустые папки или неполные файлы, но критических ошибок нет |
| КРИТИЧНО | Отсутствуют ключевые папки, много повреждённых файлов, расхождение с `ARCHITECTURE.md` |

## 📌 ПРИМЕЧАНИЯ ДЛЯ АССИСТЕНТА

- Не изменять файлы без команды. Только проверять и докладывать.
- Отчёт должен быть кратким, без лишних пояснений.
- Если обнаружена проблема — предложить способ исправления.
- При расхождении с `ARCHITECTURE.md` — указать, что именно не совпадает.
