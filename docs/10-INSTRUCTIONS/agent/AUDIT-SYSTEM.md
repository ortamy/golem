## 📋 ЭТАП 1: СКАНИРОВАНИЕ И АНАЛИЗ

Обойди рекурсивно весь проект. Собери метрики.

### Структурный анализ

- Общее количество папок
- Количество пустых папок (без файлов)
- Количество .md файлов (всего / заполненных / пустых)
- Количество не-.md файлов (.py, .json, .txt, .ps1, .html, .css)
- Топ-5 самых глубоких цепочек вложенности
- Дубликаты папок (одинаковые названия в разных местах)
- Дубликаты файлов (одинаковые имена в разных папках)
- Папки с одним файлом внутри (кандидаты на объединение)

### Контент-анализ

- Файлы с прочерками `—` в разделах ЭТИМОЛОГИЯ, КОНТЕКСТ, СВЯЗЬ, ИСКАЖЕНИЯ, СВОДКА
- Файлы без единого стиха ТаНаХа (нет ни одной строки с ивритом)
- Файлы без метаданных или с незаполненными полями аудита
- Файлы с битыми ссылками (связанные файлы, которые не существуют)
- Топ-20 самых больших файлов (по килобайтам)
- Топ-20 самых маленьких файлов (менее 500 байт — кандидаты на удаление или заполнение)
- Файлы с религионимами (прогон check-tahor.py)

### Технический анализ

- Python-скрипты в корне проекта
- Дублирующиеся скрипты в tools/ (делают одно и то же)
- Файлы без хеша в метаданных
- JSON-файлы, которые можно автосгенерировать (вместо ручного хранения)
- Скрипты без комментария с названием и путём во второй строке

### Связность

- Самые связанные файлы (на них ссылается больше всего других файлов)
- Файлы-сироты (на них никто не ссылается)
- Оборванные связи (файл ссылается на несуществующий)

## 📋 ЭТАП 2: РЕКОМЕНДАЦИИ

На основе анализа выдай список рекомендаций. Сгруппируй по категориям.

### Структурные

- Какие папки объединить (пример: две папки с 1 файлом внутри → одна папка)
- Какие папки удалить (пустые, дубликаты, устаревшие)
- Какие папки создать (напрашиваются из содержимого, но отсутствуют)
- Как упростить вложенность (тройная вложенность без необходимости)

### Контентные

- Какие файлы требуют срочного заполнения (приоритет 🔴)
- Какие файлы можно объединить (дубликаты тем, пересекающиеся исследования)
- Какие исследования отсутствуют, но напрашиваются из уже созданных
- В каких файлах нужно заменить эмодзи на иконки из web/icons/32/

### Технические

- Какие скрипты удалить (временные, неиспользуемые)
- Какие скрипты объединить (делают одно и то же)
- Что автоматизировать (ручные процессы, которые можно скриптовать)
- Где добавить проверки (в generate-скрипты, в check-скрипты)

## 📋 ЭТАП 3: РЕАЛИЗАЦИЯ

После одобрения рекомендаций человеком — выполни их.

### Порядок действий

1. Сначала — удаление (мусор, дубликаты, пустые папки)
2. Потом — объединение (файлы, папки)
3. Потом — создание (недостающие файлы, индексные файлы)
4. Потом — обновление (TECHNICAL-DEBT.md, STRUCTURE.md, ROADMAP.md)
5. В конце — проверка (прогон check-tahor.py, проверка ссылок)

### Обновление документации

- `docs/TECHNICAL-DEBT.md` — добавить новые задачи, отметить выполненные
- `docs/STRUCTURE.md` — обновить структуру после изменений
- `docs/ROADMAP.md` — обновить дорожную карту
- `docs/STATS.md` — обновить статистику проекта
- `docs/AUDIT-REPORT.md` — сохранить отчёт об аудите
