## 📋 Обзор

Модуль «Настройки / Администрирование» объединяет все точки управления проектом в едином интерфейсе. Доступен по маршруту `#admin-settings` (или `#settings` как алиас).

**Требования:**
- Только для администратора (проверка через `AccessGate.isAdmin()`)
- Гостевой режим показывает сообщение «Доступ только для администратора»

## 📑 Вкладки

### 1. Модель (`#settings-tab-model`)

Управление моделью AI по умолчанию:

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|-------------|----------|
| Модель | select | Claude Sonnet 4 | Выбор из 6 предустановленных моделей |
| Температура | range 0–2 | 0.7 | Креативность ответов |
| Max токенов | number | 4096 | Максимальная длина ответа |
| Top-P | range 0–1 | 0.95 | Nucleus sampling |
| Frequency Penalty | range 0–2 | 0.0 | Штраф за повторение |
| Presence Penalty | range 0–2 | 0.0 | Штраф за темы |

**Кнопки:**
- «Сохранить настройки модели» — сохраняет в `localStorage` (ключ `golem_settings_model`)
- «Тестовый запрос» — проверяет текущую конфигурацию

**Хранение:** localStorage, ключ `golem_settings_model`

### 2. Кэш (`#settings-tab-cache`)

Управление кэшированными данными:

| Тип кэша | Описание | Действие |
|----------|----------|----------|
| KV-кэш (localStorage) | Все записи с префиксом `golem_` | Очистить KV |
| RAG-индексы | Индексы `golem_rag_*`, `golem_knowledge_*` | Переиндексировать RAG |
| Кэш разбора слов | Кэш WordAnalyzer (`golem_wa_cache`) | Очистить кэш слов |
| Весь кэш | Все перечисленные хранилища | Очистить всё |

Каждая карточка показывает размер в B/KB/MB и количество записей.

### 3. Агенты (`#settings-tab-agents`)

Управление 7 AI-агентами:

| Агент | Модель | Управление |
|-------|--------|------------|
| Исследователь (researcher) | Claude Sonnet 4 | Вкл/Выкл + приоритет 1–10 |
| Разоблачатель (exposer) | GPT-4o | Вкл/Выкл + приоритет |
| Сборщик (collector) | Claude Haiku 3.5 | Вкл/Выкл + приоритет |
| Критик (critic) | Claude Sonnet 4 | Вкл/Выкл + приоритет |
| Семитолог (semitic) | GPT-4o | Вкл/Выкл + приоритет |
| Компаратор (comparator) | Claude Sonnet 4 | Вкл/Выкл + приоритет |
| Редактор (editor) | Claude Haiku 3.5 | Вкл/Выкл + приоритет |

Изменения сохраняются автоматически в `localStorage` (ключ `golem_settings_agents`).

### 4. Метрики (`#settings-tab-metrics`)

Мониторинг в реальном времени (обновление каждые 2 секунды):

| Метрика | Описание |
|---------|----------|
| CPU | Симулированная загрузка процессора (0–100%) |
| RAM | Использование localStorage (в B/KB/MB) |
| Записей в кэше | Количество `golem_*` записей |
| Состояние модели | Текущая выбранная модель |

Детальная таблица включает: браузер, размер localStorage, активную модель, температуру, max токенов, версию лаборатории.

### 5. Логи (`#settings-tab-logs`)

История действий в лаборатории:

- Хранится в localStorage (ключ `golem_settings_logs`)
- Лимит: до 200 записей
- Формат: время + действие
- Сортировка: свежие сверху
- **Кнопки:** «Очистить логи», «Экспорт логов (JSON)»

### 6. Ошибки (`#settings-tab-errors`)

Логи ошибок и предупреждений:

- Хранится в localStorage (ключ `golem_settings_errors`)
- Лимит: до 100 записей
- Формат: время + ошибка + источник
- Строки с ошибками подсвечены левой золотой полосой

### 7. Экспорт (`#settings-tab-export`)

Экспорт и импорт данных:

| Функция | Описание |
|---------|----------|
| Экспорт всех настроек | Скачивает `golem-settings-export.json` (модель + агенты + логи + lab-config) |
| Импорт настроек | Загрузка JSON-файла, восстановление до 4 разделов |
| Экспорт lab-config.json | Для коммита в репозиторий (гостевые токены, цвет, скрытые разделы) |

### 8. Админ (`#settings-tab-admin`)

Легаси-функции:

- **Пароль администратора** — смена локального пароля (логин: `admin`, хранится в `localStorage`)
- **Гостевые ссылки-приглашения** — создание токенов вида `#access=xxx`
- **Внешний вид** — акцентный цвет, раздел по умолчанию
- **Разделы, скрытые от гостей** — чекбоксы для 19 разделов
- **Выход** — кнопка выхода из режима администратора

## 🔧 Техническая реализация

### JavaScript
- **Файл:** `products/website/researchlab/js/admin-settings.js`
- **Объект:** `AdminSettings` (также доступен как `window.Settings`)
- **Методы:** `init()`, `render()`, `destroy()`, `logout()`, `addLog()`, `addErrorLog()`
- **Зависимости:** `AccessGate`, `LabRouter`

### CSS
- **Файл:** `products/website/researchlab/css/admin.css`
- **Тёмная и светлая темы** через `[data-theme="light"]`
- **Компоненты:** карточки кэша, карточки агентов, метрики, логи, toggle-переключатели, range-слайдеры, навигационные вкладки

### Ключи localStorage

| Ключ | Назначение |
|------|-----------|
| `golem_settings_model` | Конфигурация модели (JSON) |
| `golem_settings_agents` | Конфигурация агентов (JSON) |
| `golem_settings_logs` | Логи действий (массив JSON) |
| `golem_settings_errors` | Логи ошибок (массив JSON) |
| `golem_admin_password_override` | Пароль администратора (строка) |
| `golem_theme` | Тема: `light` / `dark` |
| `golem_wa_cache` | Кэш разбора слов |
| `golem_rag_*` | RAG-индексы |
| `golem_knowledge_*` | Индексы знаний |

### Маршруты
- `#admin-settings` — основной маршрут
- `#settings` — алиас (редиректит на `#admin-settings` через `router.js`)

## 🔄 Применение изменений

Все настройки сохраняются в `localStorage` и применяются **мгновенно, без перезапуска сервера**.

Для персистентности за пределами одного браузера:
1. На вкладке «Экспорт» нажмите «Экспорт lab-config.json»
2. Замените файл `products/website/researchlab/data/lab-config.json`
3. Закоммитьте изменения в репозиторий

## 🔌 Программный доступ

```javascript
// Чтение конфигурации модели
const modelCfg = JSON.parse(localStorage.getItem('golem_settings_model'));

// Чтение конфигурации агентов
const agentCfg = JSON.parse(localStorage.getItem('golem_settings_agents'));

// Добавление лога из любого модуля
if (window.Settings) {
  Settings.addLog('Действие выполнено');
  Settings.addErrorLog('Ошибка загрузки', 'module-name');
}
