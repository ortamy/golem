## 🔥 ОБЩАЯ КОНЦЕПЦИЯ

Мульти-агентная система «Голем» — это **набор специализированных агентов**, каждый из которых выполняет **свою роль**:

- **Исследователь** — разбирает корни, стихи, термины.
- **Редактор** — проверяет формат, метаданные, стиль.
- **Разоблачитель** — ищет подмены, сравнивает переводы.
- **Генератор идей** — предлагает новые темы, связи, доски.
- **Архитектор** — строит структуру, связи, карты.
- **Контролёр** — проверяет целостность, ссылки, дубликаты.
- **Собиратель** — объединяет результаты в единый файл.

## 🧩 АГЕНТЫ И ИХ РОЛИ

---

### 1. АГЕНТ-ИССЛЕДОВАТЕЛЬ (Researcher)

**Задача:** разбирать корни, стихи, термины по шаблону.

**Что делает:**
- Получает задание (корень, стих, термин).
- Проводит этимологический разбор (корень, палео-образы).
- Собирает убойный контекст (3–5 стихов).
- Пишет черновик исследования по шаблону.

**Вход:** корень, стих или термин.
**Выход:** черновик `.md`-файла.

---

### 2. АГЕНТ-РЕДАКТОР (Editor)

**Задача:** проверять формат, метаданные, стиль.

**Что делает:**
- Проверяет структуру файла (заголовки, разделы, эмоджи).
- Проверяет метаданные (все ли поля заполнены).
- Проверяет стиль (без таблиц, без грецизмов, иврит + транслит + перевод).
- Возвращает исправленный файл.

**Вход:** черновик `.md`-файла.
**Выход:** исправленный `.md`-файл.

---

### 3. АГЕНТ-РАЗОБЛАЧИТЕЛЬ (Exposer)

**Задача:** искать подмены в переводах и текстах.

**Что делает:**
- Сравнивает ТМ, LXX, Синодальный.
- Находит греческие и латинские философемы.
- Подсвечивает подмены.
- Предлагает ивритскую замену.

**Вход:** стих или текст.
**Выход:** список подмен с рекомендациями.

---

### 4. АГЕНТ-ГЕНЕРАТОР ИДЕЙ (Idea Generator)

**Задача:** предлагать новые темы, связи, углы.

**Что делает:**
- Анализирует существующие исследования.
- Находит незаполненные темы.
- Предлагает новые корни, стихи, доски.
- Генерирует промпты для детективных досок.

**Вход:** текущий корпус `content/`.
**Выход:** список идей с приоритетом.

---

### 5. АГЕНТ-АРХИТЕКТОР (Architect)

**Задача:** строить структуру, связи, карты.

**Что делает:**
- Анализирует структуру `content/`.
- Строит граф связей между терминами.
- Предлагает новые папки и перегруппировки.
- Генерирует карту связей (визуальную или текстовую).

**Вход:** текущая структура `content/`.
**Выход:** обновлённая структура + карта связей.

---

### 6. АГЕНТ-КОНТРОЛЁР (Controller)

**Задача:** проверять целостность, ссылки, дубликаты.

**Что делает:**
- Проверяет битые ссылки.
- Проверяет дубликаты (по корням, стихам, темам).
- Проверяет наличие метаданных.
- Проверяет соответствие структуры `ARCHITECTURE.md`.

**Вход:** весь проект.
**Выход:** отчёт об ошибках.

---

### 7. АГЕНТ-СОБИРАТЕЛЬ (Collector)

**Задача:** объединять результаты в единый файл.

**Что делает:**
- Собирает результаты всех агентов.
- Объединяет их в один документ (отчёт, сводка, дайджест).
- Генерирует итоговый файл.

**Вход:** результаты всех агентов.
**Выход:** итоговый `.md`-файл или HTML.

## 🔄 ПОТОК РАБОТЫ

```
Задача → Агент-Исследователь → черновик.md
черновик.md → Агент-Редактор → исправленный.md
исправленный.md → Агент-Разоблачитель → подмены.txt
подмены.txt + исправленный.md → Агент-Собиратель → итоговый файл

Агент-Генератор идей → Агент-Архитектор → новая структура
Агент-Контролёр → отчёт об ошибках → Агент-Собиратель
```
