## 🔄 ПОТОК ДАННЫХ

### Путь исследования
```
Идея → drafts/ → researches/ → проверка чекерами → публикация
```

### Путь термина
```
Ивритское слово → terminology/ → глоссарий → перекрёстные ссылки
```

### Полный цикл проверки одного файла
```
Новый файл
  ↓
check-naming.py          (имя корректно?)
  ↓
validate-metadata.py     (метаданные в порядке?)
  ↓
check-religionisms.py    (религионимы исправлены?)
  ↓
check-links.py           (ссылки не битые?)
  ↓
unify-metadata.py        (метаданные по шаблону?)
  ↓
sync-structure.py        (STRUCTURE.md обновлён?)
  ↓
✅ Файл готов
```

## ⚙️ КЛЮЧЕВЫЕ КОМПОНЕНТЫ

### Метаданные
Каждый md-файл содержит блок метаданных. Обязательные поля: Файл, Версия, Дата создания, Последнее обновление, Причина обновления, Статус, Тема. Дополнительные: Аудит (статус 4 проверок), Хеш (для отслеживания изменений), Связанные файлы, Проверок на религионизмы, Достоверность.

### Словари `tahor/`
Центральный источник запрещённых слов. Используются скриптами: `check-religionisms.py` для замены, `check-tahor-sync.py` для синхронизации, `check-exposure.py` для маркеров искажений.

### Кэш
`tools/cache/` хранит: `religionisms-cache.json` — скомпилированный словарь замен, `scan-cache.json` — результаты последнего сканирования, `dirty-files.json` — файлы с неисправленными религионимами, `golem-config.json` — настройки меню. Очистка: `python tools/checkers/clear-cache.py`

### Меню
`golem.py` автоматически находит все скрипты в подпапках `checkers/`, `generators/`, `reports/`, `automation/`. Новый скрипт в любой из этих папок появляется в меню без правки кода.

## 🔐 ПРИНЦИПЫ АРХИТЕКТУРЫ

1. **Единый источник истины:** словари `tahor/` — все проверки берут данные оттуда
2. **Автоматическое обнаружение:** новые скрипты автоматически появляются в меню
3. **Монолитность:** каждый md-файл — один блок, копируется одним нажатием
4. **Кэширование:** ускорение повторных проверок в 10-50 раз
5. **Многопоточность:** тяжёлые проверки используют до 8 потоков
6. **Разделение ответственности:** чекеры проверяют, генераторы создают, автоматизация управляет
