# DATA SOVEREIGNTY — Суверенитет над данными

**Файл:** `docs/02-ARCHITECTURE/DATA-SOVEREIGNTY.md`  
**Версия:** 1.0  
**Принцип:** Пользователь хранит данные у себя и решает, что и когда публиковать.

---

## 1. Принцип

В «Големе» данные не принадлежат серверу. Они принадлежат пользователю.

- **Локальное хранение** — все черновики, разборы, карточки и исследования хранятся на устройстве пользователя.
- **Публикация по выбору** — пользователь сам решает, какие данные отправлять на сервер «Голема».
- **Сервер как витрина** — сервер «Голема» не хранит ничего, кроме того, что пользователь решил опубликовать.

---

## 2. Почему это важно

- **Пользователь не зависит от сервера** — если сервер исчезает, все данные остаются у пользователя.
- **Пользователь не теряет контроль** — он сам решает, что и когда показывать.
- **Пользователь не «делится всем»** — он делится только тем, что считает готовым.

---

## 3. Как это работает

1. **Пользователь создаёт контент локально** (на своём устройстве).
2. **Контент хранится локально** — в Markdown, JSON или локальной базе.
3. **Пользователь выбирает файлы для публикации** и отправляет их на сервер.
4. **Сервер отображает только опубликованное**.

---

## 4. Что это даёт проекту

- **Суверенитет** — пользователь остаётся владельцем своих данных.
- **Безопасность** — сервер не видит черновиков и не знает, над чем работает пользователь.
- **Свобода** — публикация не является принудительной.
