## 🏛 АРХИТЕКТУРА

### Чекер (checker)
Скрипт проверки. Находит проблемы в файлах проекта. Примеры: `check-religionisms.py`, `check-links.py`, `check-duplicates.py`. Все чекеры лежат в `tools/checkers/` и имеют префикс `check-`.

### Генератор (generator)
Скрипт создания. Генерирует файлы, отчёты, книги. Примеры: `generate-book.py`, `generate-files-json.py`, `generate-knowledge-cache.py`. Все генераторы в `tools/generators/`, префикс `generate-`.

### Отчёт (report)
Скрипт аналитики. Собирает статистику, строит дашборды. Примеры: `report-dashboard.py`, `report-stats.py`. Все в `tools/reports/`, префикс `report-`.

### Автоматизация (automation)
Скрипты для автоматических действий: обновление версий, управление задачами, очистка кэша. В `tools/automation/`, префикс `auto-`.

### Утилита (utility)
Скрипты общего назначения: поиск, переименование, редактирование кода. В `tools/utils/`.

## 🧠 НЕЙРОСЕТЬ И АГЕНТ

### Нейросеть (LLM)
Модель генерации текста. Запускается через `ed-neural/inference/server.py`. Отвечает на запросы через `client.py`.

### ИИ-агент (AI agent)
Автономная программа которая сама решает какие инструменты запустить. `ed-agent/agent.py`. Отличается от ассистента тем что работает без постоянных команд.

### ИИ-ассистент (AI assistant)
Помощник в IDE. Ждёт команды, делает что сказал. Cline в VS Code. Не помнит контекст между сессиями.

### RAG (Retrieval-Augmented Generation)
Поиск релевантных документов + генерация ответа. Нейросеть ищет в кэше знаний подходящие фрагменты и отвечает на их основе.

### CAG (Cache-Augmented Generation)
Предварительно сгенерированный кэш знаний. Все документы прогоняются через нейросеть один раз, создаётся кэш. При запросе — быстрый ответ без поиска.

### Кэш знаний (knowledge cache)
JSON-файлы в `tools/cache/neural-cache/`. Содержат чанки, суммари и индекс всех исследований. Генерируются через `generate-knowledge-cache.py`.

### Чанк (chunk)
Фрагмент текста фиксированной длины (500 символов). На которые разбиваются исследования для кэша знаний.

## 📁 ФАЙЛЫ И ПУТИ

### Префикс папки
Первое слово в имени файла совпадает с названием папки. `check-*` для чекеров, `generate-*` для генераторов, `report-*` для отчётов, `auto-*` для автоматизации.

### Метаданные (metadata)
Блок в начале каждого `.md` файла между `**Метаданные файла**` и `---`. Содержит: файл, версию, дату, тему, хеш, аудит.

### Хеш (hash)
MD5-хеш содержимого файла. Обновляется автоматически при проверке через `check-religionisms.py`. Нужен для отслеживания изменений.

### Монолит (monolith)
Файл который копируется одним нажатием. Один `.md` блок без вложенных код-блоков.

## 🔧 ИНСТРУМЕНТЫ

### Code Injector
Скрипт для программного редактирования кода. Вставляет, заменяет, удаляет строки по паттернам. `tools/utils/code-injector.py`.

### Exposure
Система разоблачения. Файлы в `instructions/exposure/`. Описывают типы искажений, методы, приёмы, механизмы подмены.

### Тахор (Tahor)
Карты очищения языка. Файлы в `instructions/tahor/`. Содержат списки религионимов, грецизмов, славянизмов, латинизмов.

### Бэкап (backup)
Копия файла перед изменением. Создаётся автоматически при запуске `code-injector.py` или через `tools/backup/`.

### Кэш (cache)
Временные файлы в `tools/cache/`. Ускоряют повторные проверки. Можно сбросить через `auto-clear-cache.py`.

### Драй-ран (dry run)
Режим предпросмотра. Показывает что будет сделано без реальных изменений. Флаг `--dry-run`.

## 🔄 ПРОЦЕССЫ

### Аудит (audit)
Четырёхэтапная проверка текста: bdikah (религионизмы) → mivdak (полезность) → tikun (задачи) → factcheck (факты).

### Fine-tune
Дообучение нейросети на своих данных. Через `ed-neural/scripts/prepare-finetune-data.py` и `train.py`.

### Деплой (deploy)
Публикация веб-интерфейса на GitHub Pages. Автоматически через GitHub Actions при пуше в main.
