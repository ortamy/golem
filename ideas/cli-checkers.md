## 🔥 ЧТО ЭТО

CLI-чекеры — это скрипты (bash, python), которые работают в терминале. Ты даёшь им текст — они прогоняют его через наши принципы и выдают результат.

**Почему командная строка:**
- Быстро. Нет интерфейсов, нет задержек.
- Прозрачно. Видно, что делает скрипт.
- Скриптуемо. Можно встроить в конвейеры обработки.
- Свободно. Никакой зависимости от нейросетей и API.

**Что делают чекеры:**
1. `bdikah` — находит религионизмы, грецизмы, латинизмы
2. `mivdak` — оценивает полезность текста по 23 методам
3. `factcheck` — проверяет фактические утверждения
4. `tanakh` — возвращает ивритский стих по ссылке

## 🛠️ КОНЦЕПЦИЯ ЧЕТЫРЁХ ЧЕКЕРОВ

### 1. BDIKAH.SH — проверка на религионизмы

**Что делает:**
- Берёт текст (файл, строку, stdin)
- Прогоняет через словарь запрещённых слов
- Выдаёт список: ❌ нарушение → замена

**Словарь:** `lib/forbidden-words.txt`
Формат: `запрещённое слово|категория|замена`

Пример:
```
Бог|religionim|Элоhим
Господь|religionim|Яхве
крещение|grecism|твила
```

**Использование:**
```
./bdikah.sh text.txt
echo "Господь Бог" | ./bdikah.sh
./bdikah.sh --add "ересь|grecism|кат"
```

**Вывод:**
```
בְּדִיקָה (bdikah) — проверка текста
=====================================
❌ «Господь» — religionim. Замена: «Яхве»
❌ «Бог» — religionim. Замена: «Элоhим»
=====================================
❌ Итог: найдено 2 нарушений
```

### 2. MIVDAK.SH — аудит полезности

**Что делает:**
- Оценивает текст по 23 методам разоблачения
- Каждый метод проверяется автоматически (по ключевым словам) или требует ручной проверки
- Выдаёт итог: готов / требует доработки

**Методы (23):**
1. Этимологический удар
2. Сравнение переводов
3. Конкретизация
4. Корневые связи
5. Обратный перевод
6. Транслитерация
7. Семитский синтаксис
8. Исторический контекст
9. Плоды
10. Сад
11. Труба
12. Древнее письмо
13. Возвращение образов
14. Распознание муштры
15. Узкие места
16. Две системы
17. Язык творения
18. Карта сдвига
19. Пустыня
20. Змей
21. Ваалама
22. Колизей
23. Помазание

**Использование:**
```
./mivdak.sh research.md
```

**Вывод:**
```
מִבְדָּק (mivdak) — аудит полезности
=====================================
✅ 1. Этимологический удар — есть
❌ 2. Сравнение переводов — нет
⚠️ 3. Конкретизация — требуется ручная проверка
...
=====================================
Пройдено проверок: 15/23
✅ Текст готов к публикации
```

### 3. FACTCHECK.SH — проверка фактов

**Что делает:**
- Находит даты и проверяет их на достоверность
- Ищет ссылки на источники
- Извлекает ссылки на стихи ТаНаХа
- Отмечает гипотетические конструкции
- Выдаёт итог: сколько подтверждено, сколько гипотез, сколько опровергнуто

**Использование:**
```
./factcheck.sh text.txt
```

**Вывод:**
```
בְּדִיקַת עֻבְדּוֹת (factcheck) — проверка фактов
===================================================
📅 ДАТЫ: 325 год (Никейский собор) — ✅ подтверждено
📚 ИСТОЧНИКИ: ✅ есть ссылки на масоретский текст
📖 СТИХИ: Берешит 1:1, Шмот 20:1 — ✅
🔍 ГИПОТЕЗЫ: ⚠️ 3 маркера гипотетичности
===================================================
📊 ИТОГО: 5 подтверждено, 3 гипотезы, 1 опровергнуто
```

---

### 4. TANAKH.SH — поиск в ТаНаХе

**Что делает:**
- Ищет стих по ссылке (книга глава:стих)
- Возвращает: иврит + транслитерация + перевод
- Поддерживает поиск по ключевым словам

**Использование:**
```
./tanakh.sh "Дварим 6:4"
./tanakh.sh --search "Яхве"
```

**Вывод:**
```
📖 Дварим 6:4 — ШМА
שְׁמַע יִשְׂרָאֵל יְהוָה אֱלֹהֵינוּ יְהוָה אֶחָד
Шма Исраэль, Яхве Элоhейну, Яхве эхад
Слушай, Исраэль: Яхве — Всесильный наш, Яхве един
```

## 📚 ИСТОЧНИКИ ДАННЫХ ДЛЯ ЧЕКЕРОВ

Чекеры используют те же данные, что и ручная проверка:

```
lib/forbidden-words.txt  ←  экспорт из instructions/forbidden-words.md
lib/tanakh.db            ←  база масоретского текста (создаётся отдельно)
lib/methods.txt          ←  список 23 методов из exposure-methods.md
lib/distortions.txt      ←  типы искажений из exposure-distortions.md
```

**Синхронизация:** скрипт `sync.sh` обновляет `lib/` из `instructions/`.

## 🛡️ ВОЗВРАЩЕНИЕ

CLI-чекеры — это не замена Свидетелю. Это инструмент в его руках. Скрипт найдёт слово. Свидетель объяснит, почему оно лжёт.

Автоматизация ускоряет проверку, но не отменяет понимания.

Путь Яхве — не автоматизация. Но автоматизация помогает идти по нему быстрее.
