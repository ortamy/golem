## 📋 ПРИОРИТЕТ 1 (СРОЧНО)

**1. `check-env.py` — проверка окружения**

Проверяет, что все необходимые инструменты установлены.

Что проверяет:
- версия Python (3.6+)
- наличие git
- наличие bash (для .sh скриптов)
- наличие установленных пакетов (curses, windows-curses на Windows)
- права на запись в папки

Использование:
```bash
python check-env.py
```

Пример вывода:
```
✅ Python 3.11.5
✅ git version 2.40.0
✅ bash доступен
✅ windows-curses установлен
❌ Нет прав на запись в neural/models/
```

---

**2. `generate-index.py` — генерация индексов для папок**

Создаёт `README.md` в каждой папке со списком файлов.

Что генерирует:
- `terminology/README.md` — список всех терминов с эмодзи
- `researches/README.md` — список исследований по категориям
- `tools/README.md` — список инструментов с описанием

Использование:
```bash
python generate-index.py
python generate-index.py --dir terminology
```

---

**3. `daily-report.py` — ежедневный отчёт**

Автоматически генерирует и отправляет отчёт о состоянии проекта.

Что включает:
- количество файлов изменённых за день
- новые термины и исследования
- исправленные ошибки
- статистика метаданных

Использование:
```bash
python daily-report.py --send --telegram
```

Формат вывода: Markdown / JSON / Telegram

---

**4. `check-health.py` — проверка здоровья проекта**

Комплексная проверка всех систем проекта.

Что проверяет:
- все ли скрипты в `tools/` работают
- нет ли битых ссылок
- нет ли дубликатов
- метаданные в порядке
- имена файлов корректны

Использование:
```bash
python check-health.py
python check-health.py --fix
```

---

## 📋 ПРИОРИТЕТ 2 (ПЛАНОМЕРНО)

**5. `migrate-terms.py` — миграция терминов**

Массовое обновление терминов под новый стандарт.

Что делает:
- добавляет недостающие метаданные
- исправляет транслитерацию
- приводит заголовки к формату (эмоджи + КАПС)

Использование:
```bash
python migrate-terms.py --dry-run
python migrate-terms.py --apply
```

---

**6. `clean-cache.py` — очистка кэша**

Удаляет временные файлы и кэш.

Что удаляет:
- `__pycache__/` во всех папках
- `*.pyc`, `*.pyo`
- временные файлы `*.tmp`, `*.log`
- старые бэкапы (кроме последних 5)

Использование:
```bash
python clean-cache.py
python clean-cache.py --keep-backups 10
```

---

**7. `export-json.py` — экспорт в JSON**

Экспортирует все данные в JSON для API или веб-интерфейса.

Что экспортирует:
- все термины из `terminology/`
- все исследования из `researches/`
- структуру репозитория

Использование:
```bash
python export-json.py
python export-json.py --output web/data.json
```

---

**8. `check-versions.py` — проверка версий**

Проверяет актуальность версий во всех файлах.

Что проверяет:
- все ли файлы имеют версию
- нет ли устаревших версий
- нужно ли обновить версию

Использование:
```bash
python check-versions.py
python check-versions.py --update
```

---

## 📋 ПРИОРИТЕТ 3 (ДОЛГОСРОЧНО)

**9. `auto-commit.py` — автоматический коммит**

Автоматически коммитит изменения в Git.

Как работает:
- проверяет изменения каждые N минут
- создаёт коммит с шаблонным сообщением
- пушит в удалённый репозиторий

Использование:
```bash
python auto-commit.py --interval 30
python auto-commit.py --stop
```

Внимание: требует осторожности!

---

**10. `backup-s3.py` — бэкап в облако**

Загружает бэкап в облачное хранилище.

Поддерживаемые сервисы:
- Amazon S3
- Google Drive
- Yandex Disk
- FTP

Использование:
```bash
python backup-s3.py --provider s3
python backup-s3.py --provider google --upload
```

---

**11. `notify-telegram.py` — уведомления в Telegram**

Отправляет уведомления в Telegram бота.

Когда отправляет:
- после завершения проверок
- при ошибках
- ежедневные отчёты

Использование:
```bash
python notify-telegram.py --message "Завершено"
python notify-telegram.py --report daily
```

Требует: токен бота и chat_id в конфиге.

---

**12. `schedule-tasks.py` — планировщик задач**

Запускает задачи по расписанию (аналог cron).

Как работает:
- читает конфиг `schedule.json`
- запускает задачи в указанное время
- логирует выполнение

Пример конфига:
```json
{
  "tasks": [
    {"cmd": "python check-health.py", "time": "02:00"},
    {"cmd": "python daily-report.py", "time": "08:00"}
  ]
}
```

---

**13. `generate-sitemap.py` — карта сайта**

Генерирует sitemap.xml для веб-интерфейса.

Что включает:
- все страницы веб-интерфейса
- даты последнего обновления
- приоритеты

Использование:
```bash
python generate-sitemap.py
python generate-sitemap.py --output web/sitemap.xml
```

---

**14. `analyze-language.py` — анализ языка**

Анализирует использование религионимов и языковых конструкций.

Что анализирует:
- частоту запрещённых слов
- прогресс очистки текстов
- статистику по файлам

Использование:
```bash
python analyze-language.py
python analyze-language.py --fix --auto
```

---

**15. `merge-terms.py` — объединение терминов**

Объединяет дублирующиеся термины.

Как работает:
- находит похожие термины
- предлагает объединить
- сохраняет лучшую версию

Использование:
```bash
python merge-terms.py --dry-run
python merge-terms.py --merge
```

---

**16. `generate-idea-from-template.py` — создание идеи из шаблона**

Интерактивное создание нового файла идеи.

Что делает:
- задаёт вопросы по шаблону
- создаёт файл в `ideas/`
- заполняет метаданные

Использование:
```bash
python generate-idea-from-template.py
```
